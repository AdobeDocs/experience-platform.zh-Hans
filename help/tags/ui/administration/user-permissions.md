# 標籤的使用者許可權

>[!NOTE]
>
>Adobe Experience Platform Launch已經過品牌重塑，現在是Adobe Experience Platform中的一套資料收集技術。 因此，产品文档中的术语有一些改动。有关术语更改的综合参考，请参阅以下[文档](../../term-updates.md)。

Adobe Experience Platform中標籤的使用者許可權會透過Adobe Admin Console指派給使用者。 不同許可權集不會指派給個別使用者，而是會個別設定為產品設定檔。 接著，系統會將使用者指派給這些產品設定檔，以授予其已設定的許可權。

本指南提供標籤可用的不同型別許可權、標籤所授予存取權的功能，以及不同業務使用案例的一些基本實作策略的概觀。

>[!NOTE]
>
>如需使用Admin Console設定使用者許可權的相關步驟，請參閱的教學課程： [管理資料收集的許可權](../../../collection/permissions.md).

## 权限类型

在產品設定檔中，標籤的許可權分為四個類別：

1. 平台
1. 属性
1. 资产权限
1. 公司权限

### 平台

每個標籤屬性都有一個平台。 目前有兩個平台可供您用於標籤： Web和Mobile。 您可以使用此权限类型来限制或授予对特定类型属性的访问权限。當管理您的行動應用程式的團隊與管理您網站的團隊不同時，此功能會很有用。

### 属性

依預設，產品設定檔會授予對公司內目前和未來存在的所有屬性的存取權。 使用此許可權型別，您可以依名稱限制或授予特定現有屬性的存取權。

### 资产权限 {#property-rights}

您在UI中建立的任何標籤屬性都可以在Admin Console中使用，讓您在相同產品設定檔中將具有特定屬性許可權的屬性分組。

例如，如果指定的產品設定檔沒有屬性A1的存取權，屬於該設定檔的使用者將無法檢視或修改屬性A1內的任何設定。

如果使用者屬於某個具有屬性A1存取權的設定檔，使用者可在屬性A1內執行的動作取決於使用者從此設定檔所授予的許可權。 如果使用者擁有屬性A1的許可權，但沒有指派許可權，則他們對該屬性具有唯讀存取權。

下表概述可用的屬性許可權，及其所授予存取許可權的功能：

| 屬性權利 | 描述 |
| --- | --- |
| **开发** | 這可讓您執行下列動作：<ul><li>建立規則和資料元素</li><li>建立程式庫並在現有開發環境中建置</li><li>提交程式庫以供核准</li></ul>UI中的大多數日常任務都需要此許可權。 |
| **批准** | 這可讓您取得已提交的程式庫並建置至測試環境。 测试完成后，您还可以批准库以进行发布。 |
| **发布** | 這可讓您將核准的程式庫發佈到生產環境。 |
| **管理擴充功能** | 這可讓您執行下列動作： <ul><li>將新的擴充功能安裝至屬性</li><li>修改已安裝擴充功能的設定</li><li>刪除擴充功能</li></ul>請參閱擴充功能概觀檔案，以瞭解 [擴充功能的詳細資訊](../managing-resources/extensions/overview.md). 此角色通常属于 IT 或营销组，具体取决于您的组织。 |
| **管理環境** | 這可讓您建立和修改環境。 請參閱 [環境檔案](../publishing/environments.md) 以取得詳細資訊。 此角色通常属于 IT 组。 |

### 公司权限

公司权限适用于跨多个资产的权限。下表概述這些功能：

| 公司權利 | 描述 |
| --- | --- |
| **管理资产** | 這可讓您執行下列動作：<ul><li>建立新屬性</li><li>修改屬性層級的中繼資料和設定</li><li>刪除屬性</li></ul>管理员通常执行此角色。請參閱 [屬性檔案](companies-and-properties.md) 以取得詳細資訊。 |
| **開發擴充功能** | 授予建立及修改公司擁有的擴充功能套件的能力，包括私人發行和公開發行要求。 |
| **管理應用程式設定** | 唯有當您擁有Adobe Journey Optimizer或其他解決方案的授權，可授予行動應用程式內及推送訊息的存取權時，才能使用此功能。  這可讓您管理Experience Cloud所知道的應用程式，以及與Firebase Cloud Messaging服務和Apple推播通知服務通訊所需的推播認證。 |

## 用户总权限

個別使用者的總許可權取決於其在不同產品設定檔中的總成員資格。 如果使用者屬於多個產品設定檔，則每個設定檔的許可權會相加在一起，而不是相乘。

例如，產品設定檔A授予您屬性1的開發權利。 產品設定檔B授予您屬性2的發佈權利。 在這種情況下，您可以在屬性1中開發並在屬性2中發佈，但不能在屬性1中發佈或在屬性2中開發，因為您未被授予執行此操作的明確許可權。

## 权限方案

建立新產品設定檔時，不同的公司有不同的需求。 這些需求會因公司規模、組織結構、網站數量、參與管理標籤的人員數量等因素而異。

當您考慮建立產品設定檔並將使用者新增到其中時，以下是一些常見案例和建議起點。

### 独掌大权

如果您經營的小型公司只有一人負責所有工作，請授予此使用者所有屬性的許可權，並將上方所列的所有權利指派給他們。

### 职责分离

請考量貴組織中許多人參與標籤的情況。 您有一組人員（例如外部顧問）負責建立規則和資料元素，但您不想要讓他們存取生產環境。 在此情況下，您需確定除了IT團隊以外，沒有任何人會部署至生產環境。

若要完成此作業：

1. 為您的顧問建立帳戶，並僅授予他們開發權利。
1. 该顾问会在您设定的范围内进行生成和测试。
1. 如果顧問想要新的擴充功能或已準備好上線，貴組織的代表（擁有適當的許可權）會執行這些動作。

### 企业

企业公司可能已按地理区域划分其多个网站，每个地理区域由不同的团队负责。在这些团队中，开发和发布均由不同的人员负责。

这类似于上面的“职责分明”，不同之处在于企业是按地理区域进行组织的。例如，您可以為北美建立「開發」設定檔和「發佈」設定檔，並為歐洲建立個別的「開發」和「發佈」群組。

## 角色範例

下表提供您在組織中可能擁有的角色型別的一些範例，以及您應指派給他們的許可權：

| 角色 | 描述 | 属性 | 资产权限 | 公司权限 |
| --- | --- | --- | --- | --- |
| 经理 | 想要檢視系統中發生的情況，但應該無法進行任何變更。 | 自动包含 | (None) | (None) |
| 营销人员 | 可以安裝擴充功能並為現有屬性設定新標籤，但無法發佈至測試或生產環境。 | 自动包含 | <ul><li>开发</li><li>管理擴充功能</li></ul> | <ul><li>管理资产</li></ul> |
| 移动设备应用程序开发人员 | 負責在原生行動應用程式中實作Adobe和協力廠商解決方案。 | 自动包含 | <ul><li>开发</li><li>管理擴充功能</li></ul> | <li>管理资产</li><li>管理應用程式設定</li> |
| IT 团队 | 實際上不會修改任何標籤，但可完全控制預備和生產環境，以及其中的內容。 | 自动包含 | (None) | <ul><li>批准</li><li>发布</li><li>管理環境</li></ul> |
| 扩展开发人员 | 開發擴充功能，並可提交以進行核准，但無法發佈或新增至現有屬性。 | 自动包含 | <ul><li>开发</li></ul> | <ul><li>管理资产</li><li>開發擴充功能</li></ul> |
| 超級使用者 | 執行所有動作。 | 自动包含 | <ul><li>开发</li><li>批准</li><li>发布</li><li>管理擴充功能</li><li>管理環境</li></ul> | <ul><li>管理资产</li></ul> |
