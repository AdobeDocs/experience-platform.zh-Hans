# 细分作业开发人员指南

段作业是创建新受众段的异步进程。 它引用细分定义以及任何合并策略，控制实时客户用户档案如何合并跨用户档案片段的重叠属性。 当区段作业成功完成时，您可以收集有关区段的各种信息，例如在处理过程中可能发生的任何错误以及受众的最终大小。

本指南提供了帮助您更好地了解细分作业的信息，并包含使用API执行基本操作的示例API调用。

## 入门指南

本指南中使用的API端点是分段API的一部分。 在继续之前，请查阅分段开 [发人员指南](./getting-started.md)。

特别是，分段开 [发人员指南的入门部分](./getting-started.md#getting-started) ，包括相关主题的链接、在文档中阅读示例API调用的指南，以及成功调用任何Experience Platform API所需的标题的重要信息。

## 检索区段作业的列表

您可以通过向端点发出GET请求，检索IMS组织的所有区段作业的列表 `/segment/jobs` 信息。

**API格式**

```http
GET /segment/jobs
GET /segment/jobs?{QUERY_PARAMETERS}
```

- `{QUERY_PARAMETERS}`:(可&#x200B;*选*)添加到请求路径的参数，用于配置在响应中返回的结果。 可以包括多个参数，由&amp;符号(`&`)分隔。 以下列出了可用参数。

**查询参数**

以下是列出区段作业的可用查询参数列表。 所有这些参数都是可选的。 调用不带参数的此端点将检索组织可用的所有区段作业。

| 参数 | 描述 |
| --------- | ----------- |
| `start` | 指定返回的区段作业的起始偏移量。 |
| `limit` | 指定每页返回的区段作业数。 |
| `status` | 过滤器基于状态的结果。 支持的值为NEW、QUEUED、PROCESSING、SUCCEEDED、FAILED、CANCELLED、CANCELLED |
| `sort` | 对返回的区段作业进行订购。 以格式编写 `[attributeName]:[desc|asc]`。 |
| `property` | 过滤器细分作业并获得给定过滤器的精确匹配。 它可以采用以下任一格式编写： <ul><li>`[jsonObjectPath]==[value]` -对对象键进行筛选</li><li>`[arrayTypeAttributeName]~[objectKey]==[value]` -在数组中过滤</li></ul> |

**请求**

```shell
curl -X GET https://platform.adobe.io/data/core/ups/segment/jobs?status=SUCCEEDED \
 -H 'Authorization: Bearer {ACCESS_TOKEN}' \
 -H 'x-gw-ims-org-id: {IMS_ORG}' \
 -H 'x-api-key: {API_KEY}' \
 -H 'x-sandbox-name: {SANDBOX_NAME}'
```

**响应**

成功的响应将返回HTTP状态200，并将指定IMS组织的段作业列表为JSON。 以下响应返回IMS组织的所有成功区段作业的列表。

## 创建新区段作业

您可以通过向端点发出POST请求来创建新的区段 `/segment/jobs` 作业。

**API格式**

```http
POST /segment/jobs
```

**请求**

```shell
curl -X POST https://platform.adobe.io/data/core/ups/segment/jobs \
 -H 'Authorization: Bearer {ACCESS_TOKEN}' \
 -H 'Content-Type: application/json' \
 -H 'x-gw-ims-org-id: {IMS_ORG}' \
 -H 'x-api-key: {API_KEY}' \
 -H 'x-sandbox-name: {SANDBOX_NAME}' \
 -d '
[
  {
    "segmentId": "4afe34ae-8c98-4513-8a1d-67ccaa54bc05",
  }
]
 '
```

## 取消或删除特定区段作业

您可以通过向端点发出DELETE请求并在请求路径中提供区段作业的值来请求删 `/segment/jobs` 除指定 `id` 的区段作业。

**API格式**

```http
DELETE /segment/jobs/{SEGMENT_JOB_ID}
```

| 属性 | 描述 |
| -------- | ----------- | 
| `{SEGMENT_JOB_ID}` | 要 `id` 删除的区段作业的值。 |

**请求**

```shell
curl -X DELETE https://platform.adobe.io/data/core/ups/segment/jobs/d3b4a50d-dfea-43eb-9fca-557ea53771fd \
 -H 'Authorization: Bearer {ACCESS_TOKEN}' \
 -H 'x-gw-ims-org-id: {IMS_ORG}' \
 -H 'x-api-key: {API_KEY}' \
 -H 'x-sandbox-name: {SANDBOX_NAME}'
```

**响应**

成功的响应会返回HTTP状态204，其中包含以下信息。

```json
{
    "status": true,
    "message": "Segment job with id 'd3b4a50d-dfea-43eb-9fca-557ea53771fd' has been marked for cancelling"
}
```
