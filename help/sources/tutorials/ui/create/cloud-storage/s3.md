# 创建 [!DNL Amazon S3] UI中的源连接

Adobe Experience Platform中的源连接器提供了按计划摄取外部来源数据的功能。 本教程提供了创建 [!DNL Amazon S3]  (以下简称“ ”[!DNL S3]“)源连接器（使用平台用户界面）。

## 快速入门

本教程需要深入了解Adobe Experience Platform的以下组件：

- [[!DNL Experience Data Model (XDM)] 系统](../../../../../xdm/home.md)：Experience Platform用于组织客户体验数据的标准化框架。
   - [模式组合基础](../../../../../xdm/schema/composition.md)：了解XDM架构的基本构建基块，包括架构构成中的关键原则和最佳实践。
   - [架构编辑器教程](../../../../../xdm/tutorials/create-schema-ui.md)：了解如何使用架构编辑器UI创建自定义架构。
- [[!DNL Real-Time Customer Profile]](../../../../../profile/home.md)：根据来自多个来源的汇总数据提供统一的实时使用者个人资料。

如果您已经拥有有效的 [!DNL S3] 连接，您可以跳过本文档的其余部分，并继续阅读以下教程： [配置数据流](../../dataflow/batch/cloud-storage.md).

### 收集所需的凭据

要访问您的 [!DNL S3] 存储段时，您需要为以下凭据提供有效值：

| 凭据 | 描述 |
| ---------- | ----------- |
| `s3AccessKey` | 您的用户的访问密钥ID [!DNL S3] 桶。 |
| `s3SecretKey` | 您的密钥ID [!DNL S3] 桶。 |
| `serviceUrl` | （可选）自定义 [!DNL S3] 要连接的端点。 在以下情况下，此字段为必填： [!DNL S3] 存储段特定于区域。 的格式 `serviceUrl` 为： `https://s3.{REGION}.amazonaws.com/)`. |
| `bucketName` | 此 [!DNL S3] 存储段包含您的数据及其相应的描述性元数据。 您的 [!DNL S3] 存储段名称的长度必须介于3到63个字符之间，并且必须以字母或数字开头和结尾。 存储段名称只能包含小写字母、数字或连字符(`-`)，并且不能格式化为IP地址。 |
| `folderPath` | 中文件夹的路径 [!DNL S3] 存储您的数据的存储桶。 当用户访问受限时，此凭据是必需的。 |

有关入门的详细信息，请访问 [此AWS文档](https://aws.amazon.com/blogs/security/wheres-my-secret-access-key/).

## 连接您的 [!DNL S3] 帐户

在Platform UI中，选择 **[!UICONTROL 源]** 以访问 [!UICONTROL 源] 工作区。 此 [!UICONTROL 目录] 屏幕显示您可以为其创建帐户的各种源。

您可以从屏幕左侧的目录中选择相应的类别。 或者，您可以使用搜索选项查找要使用的特定源。

在 **[!UICONTROL 云存储]** 类别，选择 **[!UICONTROL Amazon S3]** 然后选择 **[!UICONTROL 添加数据]**.

![目录](../../../../images/tutorials/create/s3/catalog.png)

此 **[!UICONTROL 连接到Amazon S3]** 页面。 在此页上，您可以使用新凭据或现有凭据。

### 新帐户

如果您使用的是新凭据，请选择 **[!UICONTROL 新帐户]**. 在出现的输入表单上，提供名称、可选描述以及 [!DNL S3] 凭据。 完成后，选择 **[!UICONTROL 连接到源]** 然后留出一些时间来建立新连接。

![connect](../../../../images/tutorials/create/s3/connect.png)

### 现有帐户

要连接现有帐户，请选择 [!DNL S3] 要连接的帐户，然后选择 **[!UICONTROL 下一个]** 以继续。

![现有](../../../../images/tutorials/create/s3/existing.png)
