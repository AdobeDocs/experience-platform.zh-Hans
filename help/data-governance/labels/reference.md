# 数据使用标签一览表

>[!CONTEXTUALHELP]
>id="platform_policies_labeltype"
>title="标签类型"
>abstract="数据使用标签分为几类。 Adobe定义的标签包括合同标签、身份标签和敏感标签。 由贵组织定义的标签将分类为自定义标签。"
>text="See the data usage labels glossary for more information on these label types."

数据使用情况标签允许您根据应用于该数据的使用策略对数据集和字段进行分类。 Adobe Experience Platform数据管理提供了一些现成的核心数据使用标签，可用于开始对数据进行分类。

本文档概述了当前由 [!DNL Experience Platform]. 有关“数据管理”的更多信息，请参阅 [数据管理概述](../home.md).

## 合同标签

合同“C”标签用于对具有合同义务或与贵组织的数据管理策略相关的数据进行分类。

| 标签 | 定义 |
| --- | --- |
| **C1** | 只能以聚合形式从Adobe Experience Cloud导出数据，而不包含单个或设备标识符。 [更多信息...](#c1) |
| **C2** | 数据无法导出到第三方。 [更多信息...](#c2) |
| **C3** | 数据不能合并或与直接可识别信息一起使用。 [更多信息...](#c3) |
| **C4** | 数据不能用于定位任何网站或跨网站的广告或内容。 [更多信息...](#c4) |
| **C5** | 数据不能用于基于兴趣的内容或广告的跨站点定位。 [更多信息...](#c5) |
| **C6** | 数据不能用于现场广告定位。 [更多信息...](#c6) |
| **C7** | 数据不能用于内容的现场定位。 [更多信息...](#c7) |
| **C8** | 数据不能用于测量贵组织的网站或应用程序。 [更多信息...](#c8) |
| **C9** | 数据不能在数据科学工作流中使用。 [更多信息...](#c9) |
| **C10** | 数据无法用于拼合身份激活。 [更多信息...](#c10) |
| **C11** | 数据无法与区段匹配合作伙伴共享。 [更多信息...](#c11) |

## 身份标签

身份“I”标签用于对可识别或联系特定人员的数据进行分类。

| 标签 | 定义 |
| --- | --- |
| **I1** | 可识别或联系特定人员（而非设备）的直接可识别数据。 |
| **I2** | 可与任何其他数据结合使用以识别或联系特定人员的间接可识别数据。 |

## 敏感标签

敏感的“S”标签用于对您和您的组织认为敏感的数据进行分类。

一种可能被视为敏感的数据类型可能是不同类型的地理数据；但是，此类别不限于地理数据。

| 标签 | 定义 |
| --- | --- |
| **S1** | 数据指定纬度和经度，可用于确定设备的精确位置。 |
| **S2** | 可用于确定广泛定义的地理围栏区域的数据。 |
| **PSPD** | 允许的敏感个人数据(PSPD)是指Adobe按照合同规定允许您上传的数据，这些数据被视为“敏感”、“特殊类别的数据”，或适用法律使用的类似术语。 这特别不包括受保护健康信息(PHI)和其他受管理的健康数据。 |
| **RHD** | 指受保护健康信息(PHI)的数据，或者您在Adobe合同许可下上传的患者相关信息。 |

## 附录

### 合同标签详细信息

以下各节提供了与具体合同&quot;C&quot;标签的实施有关的详细信息。

#### C1 {#c1}

某些数据只能以聚合形式从Adobe Experience Cloud导出，而不包含单个或设备标识符。 例如，源自社交网络的数据。

#### C2 {#c2}

一些数据提供商在其合同中订有条款，禁止从最初收集的地方导出数据。 例如，社交网络合同通常会限制您从这些合同中收到的数据的传输。 C2标签的限制比 [C1](#c1)，这仅需要聚合和匿名数据。

#### C3 {#c3}

某些数据提供商在其合同中订有条款，禁止合并或使用具有直接可识别信息的数据。 例如，来自广告网络、广告服务器和第三方数据提供商的数据合同通常包括禁止使用具有直接可识别数据的此类数据的具体合同。

#### C4 {#c4}

C4是最严格的标签 — 它包含标签 [C5](#c5), [C6](#c6)和 [C7](#c7).

#### C5 {#c5}

如果满足以下三个条件，则会进行基于兴趣的定位或个性化：网站上收集的数据(1)用于对用户兴趣做出推论，(2)用于其他上下文，例如在其他网站或应用程序（网站外）上，(3)用于根据这些推论选择提供哪些内容或广告。

来自多个网站的数据组合（包括网站数据和网站外数据的组合或来自多个网站外来源的数据组合）称为跨网站数据。 不同的站点表示不同的上下文，因此在任何上下文中跨站点数据的使用都与原始上下文不同。 通常会收集并处理跨站点数据，以便做出有关用户兴趣的推论。 因此，对定位广告或内容使用跨站点数据通常符合基于兴趣的定位条件，无论该广告或内容是在网站上还是网站外显示。 例如，如果将网站数据与网站外数据结合使用，以选择在组织自己的网站上向用户显示哪个广告，则使用该数据将符合基于兴趣的定位条件。 再比如，将广告重定位到网站外的用户，也可能有资格进行基于兴趣的定位。

仅使用站外数据进行定位也可能符合基于兴趣的定位条件，因为网站外数据通常会收集和处理以做出有关用户兴趣的推断。

但是，仅使用网站数据定位内容或广告通常不符合基于兴趣的定位条件。 如果网站上的定位在其他情况下不符合基于兴趣的定位条件，则会被视为两个不同的标签。 具体而言，标签C6针对现场广告定位和报告，并特别针对广告选择、投放和报告，标签C7针对现场内容选择、投放和报告（现场内容定位）。

最终，您将自行解释标签以及如何使用该标签的数据。 下面提供了IAB和DAA框架供参考：

IAB:个性化。 收集和处理有关您使用此服务以随后在其他环境（如在其他网站或应用程序上）中为您个性化广告和/或内容的信息。 通常，网站或应用程序的内容会用于推断您感兴趣的内容，从而为将来选择的广告和/或内容提供参考。

数据：在线行为广告。 从特定计算机或设备收集关于Web查看行为的数据，并跨非附属网站，以便使用该数据来预测用户偏好或兴趣，以根据从这种Web查看行为推断的偏好或兴趣向该计算机或设备提供广告。

#### C6 {#c6}

广告是在主要旨在促进商品或服务销售的网站或应用程序上显示的消息或通知，包括文本和图像。 由您决定此类消息或通知的用途。 广告与现场内容分开，由标签覆盖 [C7](#c7). 具有C6标签的数据不能用于现场广告定位，包括在您组织的网站或应用程序上选择和投放广告，或用于衡量此类广告的投放和效果。 这包括使用先前收集的有关用户兴趣的现场数据来选择广告，处理有关显示了哪些广告、何时何地显示广告以及用户是否采取了与广告相关的任何操作（如选择广告或购买广告）的数据。 通常，根据用户的网站活动来推论用户的首选项，然后在网站广告定位中使用这些首选项将不符合基于兴趣的定位（也称为个性化）的条件，因为这并不符合基于兴趣的定位所需的全部三个要求。 *[有关这些要求，请参阅标签C5 。](#c5)*

最终，您将自行解释标签以及如何使用该标签的数据。 下面提供了IAB和DAA框架供参考：

IAB:3. 广告选择、投放、报告：收集信息，并与先前收集的信息相结合，为您选择和投放广告，并衡量此类广告的投放和效果。 这包括使用先前收集的关于您兴趣的信息来选择广告，处理有关哪些广告显示、显示频率、显示时间和位置，以及您是否采取了与广告相关的任何操作（例如选择广告或购买广告）的数据。 这不包括个性化，个性化是指收集和处理有关您使用此服务以随后在其他上下文（如网站或应用程序）中为您个性化广告和/或内容的信息，以便随着时间推移，这些信息会被用于收集和处理。

数据：在线行为广告不包括第一方、广告交付或广告报告或上下文广告的活动（即基于所访问网页内容的广告、消费者当前对网页的访问或搜索查询）。

#### C7 {#c7}

现场内容是旨在提供信息、教育或娱乐的文本和图像，不是为了促进商品或服务的销售而创建的。 由您来决定内容的用途，包括内容是否符合本机广告的条件。 C7标签不用于覆盖标签涵盖的现场广告 [C6](#c6). 具有C7标签的数据不能用于现场内容定位，包括在贵组织的网站或应用程序上选择和交付内容，也不能用于衡量此类内容的交付和有效性。 这包括以前收集的有关用户在选择内容中的兴趣、处理有关显示内容、显示频率或时长、显示时间和位置以及用户是否执行了与内容相关的任何操作（例如选择内容）的数据。 通常，根据用户的网站活动来推论用户的首选项，然后在网站内容定位中使用这些首选项将不符合基于兴趣的定位（也称为个性化）的条件，因为这并不符合基于兴趣的定位所需的全部三个要求。 *[有关这些要求，请参阅标签C5 。](#c5)*

最终，您将自行解释标签以及如何使用该标签的数据。 下面提供了IAB和DAA框架供参考：

IAB:4. 内容选择、交付、报告：信息的收集，以及与先前收集的信息的组合，用于选择和交付内容，并衡量此类内容的交付和有效性。 这包括使用之前收集的有关您兴趣的信息来选择内容，处理有关所显示内容、显示频率或时长、显示时间和位置，以及您是否执行了与内容相关的任何操作（例如选择内容）的数据。 这不包括个性化，个性化是指收集和处理有关您使用此服务以随后在其他上下文（如网站或应用程序）中为您个性化内容和/或广告的信息，以便随着时间推移，提供给您。

数据：在线行为广告不包括第一方、广告交付或广告报告或上下文广告的活动（即基于所访问网页内容的广告、消费者当前访问网页或搜索查询）。

#### C8 {#c8}

数据不能用于测量、了解和报告用户对贵组织网站或应用程序的使用情况。 这不包括基于兴趣的定位（跨站点定位），即关于您使用此服务以后在其他环境（例如，网站或应用程序）中为您个性化内容和/或广告的信息的收集。

#### C9 {#c9}

一些合同明确禁止数据科学使用数据。 有时，这些措辞的措辞禁止将数据用于人工智能(AI)、机器学习(ML)或建模。

#### C10 {#c10}

某些数据使用策略会限制使用拼合身份数据进行个性化。 如果区段的合并策略使用“专用图”选项，则C10标签会自动应用于区段。

#### C11 {#c11}

Adobe Experience Platform区段匹配允许您将第一方区段与隐私和同意首选项进行匹配，从而有助于丰富分析和下游分析。 C11标签表示不应在 [!DNL Segment Match] 进程。 在确定要从区段匹配中排除的数据集和/或字段并相应地添加了C11标签后，区段匹配工作流会自动强制添加标签。
