# 数据使用标签词汇表

数据使用标签允许您根据应用于该数据的使用策略对数据集和字段进行分类。 Adobe Experience Platform Data Governance提供了几个可用于开始数据分类的现成核心数据使用标签。

此文档概述了当前由[!DNL Experience Platform]提供的核心数据使用标签。 有关[!DNL Data Governance]的详细信息，请参阅[数据治理概述](../home.md)。

## 合同标签

合同“C”标签用于对有合同义务或与贵组织的数据管理政策相关的数据进行分类。

| 标签 | 定义 |
|---|---|
| **C1** | 只能以聚合形式从Adobe Experience Cloud导出数据，而不包括个人或设备标识符。 [更多信息...](#c1) |
| **C2** | 无法将数据导出到第三方。 [更多信息...](#c2) |
| **C3** | 数据不能与直接可识别信息组合或以其他方式使用。 [更多信息...](#c3) |
| **C4** | 数据不能用于定位任何现场或跨站点广告或内容。 [更多信息...](#c4) |
| **C5** | 数据不能用于基于兴趣的内容或广告的跨站点定位。 [更多信息...](#c5) |
| **C6** | 数据不能用于现场广告定位。 [更多信息...](#c6) |
| **C7** | 数据不能用于内容的现场定位。 [更多信息...](#c7) |
| **C8** | 数据不能用于测量您组织的网站或应用程序。 [更多信息...](#c8) |
| **C9** | 数据不能用于数据科学工作流。 [更多信息...](#c9) |
| **C10** | 数据不能用于拼接标识激活。 [更多信息...](#c10) |

## 标识标签

标识“I”标签用于对能够识别或联系特定人的数据进行分类。

| 标签 | 定义 |
|---|---|
| **I1** | 能够识别或联系特定人而不是设备的直接识别数据。 |
| **I2** | 可与任何其他数据结合使用的间接可识别数据，以识别或联系特定人。 |

## 敏感标签

敏感的“S”标签用于对您和您的组织认为敏感的数据进行分类。

您可能认为敏感的一种数据类型可能是不同类型的地理数据；但是，此类别并不局限于地理数据。

| 标签 | 定义 |
|---|---|
| **S1** | 数据，指定可用于确定设备精确位置的纬度和经度。 |
| **S2** | 可用于确定广义地理区域的数据。 |

### 合同标签详细信息

以下各节提供与具体合同&quot;C&quot;标签的实施有关的详细信息。

#### C1 {#c1}

某些数据只能以聚合形式从Adobe Experience Cloud导出，而不包括单个或设备标识符。 例如，源自社交网络的数据。

#### C2 {#c2}

一些数据提供者在其合同中有条款禁止从最初收集数据的地方出口数据。 例如，社交网络合同通常会限制您从他们那里接收的数据的传输。 C2标签比[C1](#c1)更具限制性，后者只需要聚合和匿名数据。

#### C3 {#c3}

一些数据提供商在其合同中有条款禁止将数据与直接可识别的信息合并或使用。 例如，从广告网络、广告服务器和第三方数据提供商获得的数据的合同通常包括对使用具有直接可识别数据的此类数据的具体合同禁令。

#### C4 {#c4}

C4是限制性最强的标签 — 它包含标签[C5](#c5)、[C6](#c6)和[C7](#c7)。

#### C5 {#c5}

如果满足以下三个条件，则会发生基于兴趣的定位或个性化：现场收集的数据用于(1)对用户的兴趣做出推论，(2)用于其他上下文，例如在另一个站点或应用上（非现场），和(3)用于根据这些推论选择提供哪些内容或广告。

来自多个站点的数据的组合，包括现场数据和非现场数据的组合或来自多个非现场源的数据的组合，称为跨站点数据。 不同的站点表示不同的上下文，因此在任何上下文中使用跨站点数据都与原始数据不同。 跨站点数据通常会被收集和处理，以推论用户的兴趣。 因此，对定位广告或内容使用跨站点数据通常符合基于兴趣的定位条件，而不管该广告或内容是在现场还是场外显示。 例如，如果将现场数据与场外数据结合使用来选择在组织自己的网站上向用户显示哪个广告，则该使用将符合基于兴趣的定位。 另一个示例是，将广告重新定位到网站外的用户也可能符合基于兴趣的定位。

仅使用非现场数据进行定位也可能符合基于兴趣的定位，因为非现场数据通常被收集和处理以推断用户的兴趣。

但是，仅使用现场数据定位内容或广告通常不符合基于兴趣的定位。 如果网站定位在其他情况下不符合基于兴趣的定位，则视为两个不同的标签。 具体而言，标签C6针对现场广告定位和报告，特别针对广告选择、投放和报告，标签C7针对现场内容选择、投放和报告（现场内容定位）。

最终，标签的解释以及如何强制使用带有该标签的数据取决于您。 为参考，IAB和DAA框架如下：

IAB:个性化。 收集和处理有关您使用此服务的信息，以随后在其他上下文（例如在其他网站或应用程序）为您个性化广告和/或内容。 通常，网站或应用程序的内容用于推论您的兴趣，这些兴趣会为将来选择广告和/或内容提供信息。

DAA:在线行为广告。 从特定计算机或设备收集关于Web浏览行为的数据，并随时间跨非附属网站进行，以便使用这些数据来预测用户偏好或兴趣，根据从Web浏览行为推断出的偏好或兴趣向该计算机或设备投放广告。

#### C6 {#c6}

广告是主要旨在促进商品或服务销售的网站或应用程序中显示的消息或通知，包括文本和图像。 由您决定此类消息或通知的用途。 广告与现场内容分开，由标签[C7](#c7)覆盖。 具有C6标签的数据不能用于现场广告定位，包括选择和投放贵组织网站或应用程序上的广告，也不能用于衡量此类广告的投放和效果。 这包括使用先前收集的有关用户兴趣的现场数据来选择广告、处理有关显示广告的数据、显示广告的时间和位置以及用户是否采取了与广告相关的任何操作（如选择广告或购买）。 通常，根据用户的在线活动推断用户的偏好，然后在在线广告定位中使用这些偏好，将不符合基于兴趣的定位（也称为个性化），因为它无法满足基于兴趣的定位所需的全部三个要求。 *[有关这些要求，请参阅标签C5。](#c5)*

最终，标签的解释以及如何强制使用带有该标签的数据取决于您。 为参考，IAB和DAA框架如下：

IAB:3. 广告选择、投放、报告:收集信息，并与先前收集的信息相结合，为您选择和投放广告，并衡量此类广告的投放和效果。 这包括使用先前收集到的关于您的兴趣的信息来选择广告、处理有关显示广告的数据、显示频率、显示时间和位置，以及您是否采取了与广告相关的任何操作，包括选择广告或购买。 这不包括个性化，即收集和处理有关您使用此服务以随后在其他上下文（如网站或应用程序）中为您个性化广告和/或内容的信息。

DAA:在线行为广告不包括第一方、广告投放或广告报告或情境广告的活动(即基于所访问网页内容的广告、消费者当前访问网页或搜索查询)。

#### C7 {#c7}

现场内容是旨在通知、教育或娱乐的文本和图像，并非为了促进商品或服务的销售而创建。 由您决定内容的用途，包括内容是否符合本地广告的资格。 C7标签不用于覆盖现场广告，这些广告由标签[C6](#c6)覆盖。 带有C7标签的数据不能用于现场内容定位，包括选择和投放贵组织网站或应用程序中的内容，也不能用于衡量此类内容的投放和有效性。 这包括先前收集的有关用户对所选内容的兴趣的信息、处理有关显示内容的数据、显示频率或时间、显示时间和位置的信息，以及用户是否对与内容相关的任何操作（例如，选择内容）执行了任何操作。 通常，根据用户的在线活动推断用户的偏好，然后在在线内容定位中使用这些偏好，将不符合基于兴趣的定位（也称为个性化），因为它无法满足基于兴趣的定位所需的全部三个要求。 *[有关这些要求，请参阅标签C5。](#c5)*

最终，标签的解释以及如何强制使用带有该标签的数据取决于您。 为参考，IAB和DAA框架如下：

IAB:4. 内容选择、投放、报告:信息的收集，以及与先前收集的信息的组合，用于选择和提供您的内容，并衡量此类内容的投放和有效性。 这包括使用先前收集的有关您的兴趣的信息选择内容、处理有关显示内容、显示频率或时间、显示时间和位置的数据，以及您是否对内容执行了任何相关操作（包括选择内容）。 这不包括个性化，即收集和处理有关您使用此服务以后在其他上下文（如网站或应用程序）中为您个性化内容和/或广告的信息。

DAA:在线行为广告不包括第一方、广告投放或广告报告或情境广告的活动(即基于所访问网页内容的广告、消费者当前访问网页或搜索查询)。

#### C8 {#c8}

数据不能用于衡量、了解和报告用户对贵组织网站或应用程序的使用情况。 这不包括基于兴趣的定位（跨站点定位），即有关您使用此服务以随后在其他上下文（例如网站或应用程序）中为您个性化内容和/或广告的信息的收集。

#### C9 {#c9}

有些合同明确禁止数据科学使用数据。 有时，这些术语的措辞禁止将数据用于人工智能(AI)、机器学习(ML)或建模。

#### C10 {#c10}

某些数据使用策略会限制使用拼接标识数据进行个性化。 如果区段的合并策略使用“专用图”选项，则C10标签会自动应用于区段。
