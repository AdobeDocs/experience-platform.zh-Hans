## 仪表板 {#dashboards}

Adobe Experience Platform提供多个 [!DNL dashboards] 通过它可以查看有关您组织的数据的重要信息，如在每日快照期间捕获的数据。

### 帐户配置文件功能板

帐户配置文件仪表板显示来自营销渠道的多个来源以及贵组织当前用于存储客户帐户信息的不同系统的统一帐户信息的快照。

**更新的功能**

| 功能 | 描述 |
| --- | --- |
| 按行业小组件划分的总帐户数 | 此构件显示单个指标中的帐户总数，并使用圆环图说明构成总数的行业中计数的比例大小。 |
| 已添加帐户配置文件 | 此构件使用颜色编码的条形图来说明在给定时间段内添加到帐户的用户档案计数，以及构成这些添加的用户档案的不同行业比例。 |

{style="table-layout:auto"}

请参阅 [Real-Time CDP， B2B版本概述](../../rtcdp/b2b-overview.md) 以了解有关可用B2B功能的更多信息，或者 [端到端教程](../../rtcdp/b2b-tutorial.md) 详细了解如何在B2B工作流中创建帐户配置文件。

有关可用于可视化您的帐户配置文件相关量度的小组件的更多信息，请参阅 [帐户配置文件小组件文档](../../dashboards/guides/account-profiles.md#standard-widgets).

### 配置文件仪表板

配置文件仪表板显示贵组织在Experience Platform的配置文件存储区中拥有的属性（记录）数据的快照。

**更新的功能**

| 功能 | 描述 |
| --- | --- |
| 映射的受众小组件 | 此构件显示可从用户档案仪表板下拉列表中激活到所选目标的已映射受众总数。 |

有关“用户档案”仪表板的详细信息，请参阅 [配置文件功能板概述](../../dashboards/guides/profiles.md).

### 目标仪表板

目标仪表板显示贵组织在Experience Platform中启用的目标快照。

**更新的功能**

| 功能 | 描述 |
| --- | --- |
| 受众构件 | 此构件根据应用于用户档案数据的所选合并策略，提供准备激活的区段总数。 |

{style="table-layout:auto"}

要了解有关“目标”功能板的更多信息，请参阅 [目标仪表板概述](../../dashboards/guides/destinations.md).

**新增功能**

| 功能 | 描述 |
| --- | --- |
| 通过Adobe Admin Console进行权限管理 | 现在，通过Adobe Admin Console的Adobe Experience Platform数据收集卡下方，可访问数据收集功能。 请参阅指南，网址为 [数据收集权限](../../collection/permissions.md) 以了解更多信息。<br><br>现在，数据流的权限也可通过在Adobe Experience Platform的信息卡下Admin Console进行管理，与以前为每个用户手动设置这些权限的方法相比，安全性更高。 |

欲知更多信息，请参见 [数据收集概述](../../collection/home.md).

## Experience Data Model (XDM) {#xdm}

XDM 是一种开源规范，可为导入 Adobe Experience Platform 的数据提供通用结构和定义（架构）。通过遵守 XDM 标准，所有客户体验数据都可以合并到一个通用的呈现中，以更快、更加集成的方式提供见解。您可以从客户行为中获得有价值的见解，通过区段定义客户受众，并使用客户属性实现个性化目的。

**新增功能**

| 功能 | 描述 |
| --- | --- |
| 医疗保健行业数据模型 | 引入标准医疗保健数据模型，以支持与增加数字获取、改善项目注册和推广药物信息相关的五个常见行业用例。 请参见 [医疗保健数据模型](../../xdm/schema/industries/healthcare.md) 有关这些用例以及支持它们的标准XDM组件的更多信息。<br><br>新的行业过滤器已添加到 [!UICONTROL 架构] UI，可帮助您在构建自定义架构时浏览与医疗保健相关的组件。 |

## Real-Time Customer Data Platform B2B 版 {#b2b}

Real-Time CDP B2B 版本基于 Real-Time Customer Data Platform (Real-Time CDP) 构建，专为采用业务对业务服务模式运营的营销人员而构建。它将来自多个来源的数据汇集在一起&#x200B;，并将其组合成人员和帐户配置文件的单一视图。这种统一的数据使营销人员能够精确定位特定受众，并通过所有可用渠道吸引这些受众。

**新增功能**

| 功能 | 描述 |
| --- | --- |
| 导致账户匹配 | 通过商机帐户匹配，可将已知人员配置文件联接到帐户配置文件。 然后，您可以在B2B上下文（如帐户或商机）中对数据进行分段和定位。 每天运行的作业同时使用确定性因素和概率因素将尚未与任何帐户关联的人员配置文件与最佳匹配帐户进行匹配。 然后，您可以决定是否将此类匹配项包含在区段定义中。 <br><br>欲了解更多信息，请参阅以下文档： [商机帐户匹配](../../rtcdp/b2b-ai-ml-services/lead-to-account-matching.md). 有关如何配置销售线索与帐户匹配的说明，请参阅 [帐户配置文件UI指南](../../rtcdp/account/../accounts/account-profile-ui-guide.md#configure-lead-to-account-matching).</li> |
| 预测性商机和客户评分 | 预测商机和帐户评分使用基于树的（随机林/梯度提升）机器学习方法，其中涉及从机会阶段转化事件中进行学习和预测，并将人员活动聚合到帐户级别以产生帐户分数。 此外，汇总和单位级别均提供主要影响因素，以帮助B2B营销人员更好地了解哪些元素驱动了分数。 <br><br>欲了解更多信息，请参阅以下文档： [预测性商机和客户评分](../../rtcdp/b2b-ai-ml-services/predictive-lead-and-account-scoring.md). 有关如何管理得分的信息，请参阅 [在Real-time Customer Data Platform B2B版本中管理预测性商机和客户评分。](../../rtcdp/b2b-ai-ml-services/manage-predictive-lead-and-account-scoring.md) |

有关如何监控用户档案扩充的指南，请参阅上的文档 [在UI中监控配置文件扩充](../../dataflows/ui/b2b/monitor-profile-enrichment.md). 要了解有关Real-Time CDP B2B版本的更多信息，请参阅 [Real-Time CDP B2B概述](../../rtcdp/overview.md).

## 实时客户配置文件 {#profile}

Adobe Experience Platform 使您能够为客户提供协调、一致且相关的体验，无论他们何时何地与您的品牌互动均是如此。利用实时客户配置文件，您可以看到每个客户的整体视图，其中结合来自多个渠道的数据，包括在线、离线、CRM 和第三方数据。配置文件允许您将客户数据整合到一个统一视图中，并为每一次客户交互提供可操作的、有时间戳的描述。

| 功能 | 描述 |
| ------- | ----------- |
| 孤立的配置文件边缘属性清理（限量发布） | 如果贵组织有权访问此功能，则配置文件服务现在每天都会删除用户活动区域的剩余边缘属性，以便更准确地表示您系统中的配置文件。 此清理在给定配置文件的所有配置文件片段删除后发生，并且应该影响从以下数据集合并的配置文件： `com_adobe_aep_profile_region_dataset` 标记为true。 这可能显示在许可证使用情况仪表板中的“可寻址受众”量度下降，也可能显示在配置文件仪表板中的“配置文件计数”量度下降，因为这些量度包含此版本之前的剩余边缘属性片段。 |

{style="table-layout:auto"}

要详细了解实时客户配置文件，包括使用配置文件数据的教程和最佳实践，请首先阅读[实时客户配置文件概述](../../profile/home.md)。

## 源 {#sources}

Adobe Experience Platform可以从外部源摄取数据，同时允许您使用Platform服务来构建、标记和增强这些数据。 您可以从各种来源获取数据，例如 Adobe 应用程序、基于云的存储、第三方软件和 CRM 系统。

Experience Platform 提供 RESTful API 和交互式 UI，可让您轻松为各种数据提供者设置源连接。这些源连接允许您验证并连接到外部存储系统和 CRM 服务、设置运行摄取操作的时间以及管理数据摄取吞吐量。

**新增功能**

| 功能 | 描述 |
| --- | --- |
| [!DNL Azure Data Explorer] 源全面可用 | 使用AzureData Explorer源从 [!DNL Azure] 实例到Experience Platform。 请参阅 [[!DNL Azure Data Explorer] 源概述](../../sources/connectors/databases/data-explorer.md) 以了解更多信息。 |
| 正式发布 [!DNL Generic OData] 源 | 使用 [!DNL Generic OData] 来源将支持开放数据协议的系统资源引入到Experience Platform中。 请参阅 [[!DNL Generic OData] 源概述](../../sources/connectors/protocols/odata.md) 以了解更多信息。 |
| 支持自动检测源文件属性 [!DNL Data Landing Zone] 在Experience PlatformUI中 | 此 [!DNL Data Landing Zone] 在使用Experience PlatformUI时，源现在支持自动检测文件属性。 请参阅相关文档 [创建 [!DNL Data Landing Zone] 源连接](../../sources/tutorials/ui/create/cloud-storage/data-landing-zone.md) 以了解更多信息。 |
