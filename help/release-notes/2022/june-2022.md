## [!DNL Data Science Workspace] {#dsw}

数据科学工作区使用机器学习和人工智能从您的数据中释放见解。 数据科学工作区集成到Adobe Experience Platform中，可帮助您跨Adobe解决方案使用内容和数据资源做出预测。 Data Science Workspace实现此目标的一种方式是使用JupyterLab。 JupyterLab是基于Web的用户界面，用于 <a href="https://jupyter.org/" target="_blank">Jupyter项目</a> 并紧密集成到Adobe Experience Platform中。 它为数据科学家提供了交互式开发环境，以使用Jupyter笔记本、代码和数据。

| 功能 | 描述 |
| --- | --- |
| JupyterLab启动器 | JupyterLab Lab Launcher现在包含Spark 3.2笔记本的入门产品。 Spark 2.4笔记本启动程序现已由Spark 3.2笔记本取代，并将成为此版本的一部分。 |
| Spark 3.2 | 新的Scala (Spark)和PySpark配方现在使用Spark 3.2 |
| 内核 | Scala (Spark)笔记本现在通过Scala内核编写。 PySpark笔记本现在通过Python内核编写。 已弃用Spark和PySpark内核，并设置为在后续版本中删除。 |
| 配方 | 新的PySpark和Spark配方现在遵循Docker工作流，类似于Python和R配方。 |

{style="table-layout:auto"}

有关数据科学工作区的更多常规信息，请参阅 [概述文档](../../data-science-workspace/home.md).

## 数据收集 {#data-collection}

Adobe Experience Platform提供了一套技术，可让您收集客户端客户体验数据并将该数据发送到Adobe Experience Platform Edge Network，可在其中扩充和转换数据，并将其分发到Adobe或非Adobe目标。

**新增功能**

| 功能 | 描述 |
| --- | --- |
| [数据流的访问类型配置](../../edge/datastreams/overview.md#create) | 创建新数据流时，您现在可以选择希望Edge Network接受的请求类型： <ul><li>**[!UICONTROL 混合身份验证]**：选择此选项时，边缘网络会同时接受经过身份验证的请求和未经过身份验证的请求。 当您计划使用Web SDK时选择此选项，或者 [移动SDK](https://aep-sdks.gitbook.io/docs/)，以及 [服务器API](../../server-api/overview.md). </li><li>**[!UICONTROL 仅已验证]**：选择此选项时，边缘网络仅接受经过身份验证的请求。 当您计划仅使用服务器API并希望阻止任何未经身份验证的请求被处理时，请选择此选项 [!DNL Edge Network]. </li></ul> |
| [渲染建议](../../edge/personalization/rendering-personalization-content.md#applypropositions) 而不增加量度。 | 新添加的 `applyPropositions` 命令允许您从呈现或执行建议数组 [!DNL Target] 单页应用程序，而不增加 [!DNL Analytics] 和 [!DNL Target] 量度。 这提高了报告的准确性。 |
| [移动到Web和跨域ID共享](../../edge/identity/id-sharing.md) | Adobe Experience Platform Web SDK现在支持访客ID共享功能，使您能够更准确地在移动应用程序和移动Web内容之间以及跨域提供个性化体验。 |
| [Google Data Layer标记扩展](../../tags/extensions/client/google-data-layer/overview.md) | Google Data Layer扩展允许您在标记实施中使用Google数据层。 |
| [Google Ads增强型转化事件转发扩展](https://partners.adobe.com/exchangeprogram/experiencecloud/exchange.details.108630.html) | Google Ads增强型转化扩展允许您实时增强Google Ads转化。 |
| [Mailchimp事件转发扩展](../../tags/extensions/server/mailchimp/overview.md) | Mailchimp事件转发扩展将事件发送到Mailchimp营销API，这可以触发Mailchimp营销活动、历程或交易的电子邮件。 |

欲知更多信息，请参见 [数据收集概述](../../collection/home.md).

## 查询服务 {#query-service}

查询服务允许您使用标准SQL在Adobe Experience Platform中查询数据 [!DNL Data Lake]. 您可以从以下位置连接任何数据集 [!DNL Data Lake] 并将查询结果捕获为新数据集，以用于报表、数据科学工作区或将其摄取到实时客户档案中。

**更新的功能**

| 功能 | 描述 |
| --- | --- |
| 临时架构标签 | 通过将标签应用于通过查询服务CTAS查询自动生成的临时架构的数据字段来管理对敏感数据的访问。 您可以限制使用临时架构的某些字段或数据集来控制对敏感个人数据和个人身份信息的访问。 通过使用基于属性的访问控制功能，您可以通过Platform UI为临时架构字段添加标签。 |
| `FLATTEN` 设置 | 通过第三方BI工具连接到数据库时， `FLATTEN` 设置拼合嵌套数据结构以使其成为单独的列，其中属性名称成为保存行值的列名称。 这提高了临时架构的可用性，并减少了在不支持嵌套数据结构的BI工具中检索、分析、转换和报告数据所需的工作量。 |

{style="table-layout:auto"}

有关查询服务的详细信息，请参阅 [查询服务概述](../../query-service/home.md).

## 源 {#sources}

Adobe Experience Platform可以从外部源摄取数据，同时允许您使用Platform服务来构建、标记和增强这些数据。 您可以从各种来源(如Adobe应用程序、基于云的存储、第三方软件和您的CRM系统)中摄取数据。

Experience Platform提供RESTful API和交互式UI，可让您轻松为各种数据提供程序设置源连接。 这些源连接允许您进行身份验证并连接到外部存储系统和CRM服务，设置引入运行的时间，以及管理数据引入吞吐量。

| 功能 | 描述 |
| --- | --- |
| 的Beta版 [!DNL Mixpanel] 源 | 您现在可以使用 [!DNL Mixpanel] 从中摄取分析数据的源 [!DNL Mixpanel] 帐户到Experience Platform。 请参阅 [[!DNL Mixpanel] 源文档](../../sources/connectors/analytics/mixpanel.md) 了解更多信息。 |

要了解有关源的更多信息，请参阅 [源概述](../../sources/home.md).
