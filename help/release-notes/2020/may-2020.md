## 用户界面更新 {#ux}

Adobe Experience Platform将发布对域和标题栏的更新，以改进您的体验并与其他内容相结合 [!DNL Experience Cloud] 应用程序。

- 更轻松地在组织之间切换或切换到其他应用程序
- 改进了用户帮助，包括“帮助”菜单中的精选文章和与上下文相关的文档
- 能够提供有关 [!DNL Experience Platform] 和文件支持票证

新体验的推出是逐步的。 您可以在 [https://experience.adobe.com/platform](https://experience.adobe.com/platform).

## [!DNL Data Science Workspace] {#dsw}

[!DNL Data Science Workspace] 使用机器学习和人工智能从数据中释放洞察信息。 集成到Adobe Experience Platform, [!DNL Data Science Workspace] 可帮助您跨Adobe解决方案使用内容和数据资产进行预测。 方法之一 [!DNL Data Science Workspace] 通过使用 [!DNL JupyterLab]. [!DNL JupyterLab] 是 [[!DNL Project Jupyter]](https://jupyter.org/) 与Adobe Experience Platform紧密结合。 它为数据科学家提供了一个交互式开发环境，以便与之合作 [!DNL Jupyter] 笔记本、代码和数据。

**新增功能**

| 功能 | 描述 |
|--- | ---|
| [!DNL JupyterLab Launcher] | 的 [!DNL JupyterLab Launcher] 现在包括起始项 [!DNL Spark] 2.4笔记本。 [!DNL Spark] 2.3笔记本开头现在标记为已弃用，并设置为在后续版本中删除。 |
| [!DNL Spark] 2.4 | 新扇形([!DNL Spark])和PySpark配方现在使用 [!DNL Spark] 2.4。 |
| 内核 | 斯卡拉([!DNL Spark])笔记本现在通过Scala内核进行创作。 PySpark笔记本现在通过 [!DNL Python] 内核。 的 [!DNL Spark] 和PySpark内核已弃用，并设置为在后续版本中删除。 |
| 方法 | 新PySpark和 [!DNL Spark] 现在，菜谱就跟在后面 [!DNL Docker] 类似于 [!DNL Python] 和R菜谱。 |

有关迁移笔记本和方法以使用的更多信息 [!DNL Spark] 2.4，请参见 [笔记本数据访问指南](../../data-science-workspace/jupyterlab/access-notebook-data.md). 有关数据科学工作区的更多常规信息，请参阅 [概述文档](../../data-science-workspace/home.md).

## [!DNL Real-Time Customer Profile] {#profile}

Adobe Experience Platform使您能够为客户在何处或何时与您的品牌进行交互，从而提供协调、一致的相关体验。 使用 [!DNL Real-Time Customer Profile]，您可以看到每个客户的整体视图，该视图将来自多个渠道的数据（包括在线、离线、CRM和第三方数据）进行合并。 [!DNL Profile] 允许您将不同的客户数据整合到统一视图中，为每次客户互动提供一个可操作且带有时间戳的帐户。

**新增功能**

| 功能 | 描述 |
| -----------| ---------- |
| 新配置文件导出量度 | 向配置文件导出作业添加了量度，这些作业显示了每个命名空间中已导出的配置文件总数和配置文件数。 |
| 新的可观性洞察量度 | 可观察性分析API现在具有以下用于将摄取流式引入的量度 [!DNL Profile]:传入请求速率、成功摄取速率、摄取的记录大小。 |
| 批量GET端点 | 向添加了批量GET端点 [!DNL Real-Time Customer Profile] API，用于在单个API调用中检索多个结果。 现在，您最多可以批量GET区段定义、区段作业和合并策略的100个ID。 |
| 按身份浏览配置文件 | 在 [!DNL Platform] UI您现在可以选择身份命名空间并提供身份值，以浏览配置文件。 |

## 源 {#sources}

Adobe Experience Platform可以从外部源摄取数据，同时允许您使用 [!DNL Platform] 服务。 您可以从各种来源摄取数据，如Adobe应用程序、基于云的存储、第三方软件和您的CRM系统。

[!DNL Experience Platform] 提供了RESTful API和交互式UI，让您能够轻松为各种数据提供程序设置源连接。 这些源连接允许您验证并连接到外部存储系统和CRM服务，设置摄取运行的时间，以及管理数据摄取吞吐量。

**新增功能**

| 功能 | 描述 |
| ------- | ----------- |
| 云存储系统的其他API和UI支持 | 新的源连接器 [!DNL Azure File Storage]. |
| 对数据库的其他API和UI支持 | 新的源连接器 [!DNL Azure Data Explorer]、 IBM DB2和 [!DNL Oracle DB]. |
| Adobe Audience Manager到 [!DNL Experience Platform] 数据共享 | 的预配过程 [!DNL Audience Manager] 连接器已更新。 [!DNL Audience Manager] 数据集 [!DNL Real-Time Customer Profile] 现在默认处于禁用状态。 您可以手动选择要提升到的数据集 [!DNL Profile]. 新的默认设置不具有追溯性，而且只会影响对新的 [!DNL Audience Manager] 连接器。 请参阅 [Datasets用户指南](../../catalog/datasets/user-guide.md). |
