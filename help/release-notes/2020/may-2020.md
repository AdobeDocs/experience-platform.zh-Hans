## 用户界面更新 {#ux}

Adobe Experience Platform is releasing updates to the domain and header bar to improve your experience and unify with other [!DNL Experience Cloud] applications.

- 更轻松地在组织之间或切换到其他应用程序
- 改进了用户帮助，包括“帮助”菜单中的特色文章和上下文相关文档
- Ability to give feedback about the [!DNL Experience Platform] and file support tickets

新体验的推出是渐进的。 您可以在https://experience.adobe.com/platform上视图 [体验](https://experience.adobe.com/platform)。

## [!DNL Data Science Workspace] {#dsw}

[!DNL Data Science Workspace] 使用机器学习和人工智能从数据中释放洞察。 集成到Adobe Experience Platform中 [!DNL Data Science Workspace] ，帮助您跨Adobe解决方案使用内容和数据资产进行预测。 实现这一点 [!DNL Data Science Workspace] 的方法之一是使用 [!DNL JupyterLab]。 [!DNL JupyterLab] 是基于Web的用户界面， <a href="https://jupyter.org/" target="_blank">[!DNL Project Jupyter]</a> 并且紧密集成到Adobe Experience Platform中。 它为数据科学家提供了交互式开发环境，使他们能 [!DNL Jupyter] 够处理笔记本、代码和数据。

**新增功能**

| 功能 | 描述 |
|--- | ---|
| [!DNL JupyterLab Launcher] | 现在 [!DNL JupyterLab Launcher] 包括2.4笔 [!DNL Spark] 记本的入门。 [!DNL Spark] 2.3笔记本起始页现在标记为已弃用，并设置为在后续版本中删除。 |
| [!DNL Spark] 2.4 | 新的Scala([!DNL Spark])和PySpark菜谱现在 [!DNL Spark] 使用2.4。 |
| 内核 | Scala([!DNL Spark])笔记本现在通过Scala内核创作。 PySpark笔记本现在通过内核进行 [!DNL Python] 创作。 已 [!DNL Spark] 弃用和PySpark内核，并将其设置为在后续版本中删除。 |
| 菜谱 | 新的PySpark和 [!DNL Spark] 菜谱现在遵循与 [!DNL Docker] R和菜谱类 [!DNL Python] 似的工作流程。 |

有关将笔记本和菜谱迁移至使用2.4的 [!DNL Spark] 更多信息，请参阅笔记本 [电脑迁移指南](../../data-science-workspace/recipe-notebook-migration.md)。 有关数据科学工作区的更多一般信息，请参阅 [概述文档](../../data-science-workspace/home.md)。

## [!DNL Real-time Customer Profile] {#profile}

Adobe Experience Platform使您能够为客户提供协调、一致和相关的体验，无论客户在何处或何时与您的品牌互动。 您 [!DNL Real-time Customer Profile]可以看到每个客户的整体视图，该渠道组合了多个的数据，包括在线、离线、CRM和第三方数据。 [!DNL Profile] 允许您将不同的客户数据整合到统一的视图中，为每次客户互动提供一个具有可操作性、时间戳记的帐户。

**新增功能**

| 功能 | 描述 |
| -----------| ---------- |
| 新的用户档案导出指标 | 为用户档案导出作业添加了指标，这些指标显示每个命名空间中导出的用户档案总数和用户档案数。 |
| 新的可观性洞察指标 | 可观性洞察API现在具有以下用于向中流式接收的指标 [!DNL Profile]: 传入请求速率、成功摄取速率、摄取的记录大小。 |
| 批量GET端点 | 为API添加了批量 [!DNL Real-time Customer Profile] GET端点，以在单个API调用中检索多个结果。 您现在可以批量GET（最多100个ID），用于区段定义、区段作业和合并策略。 |
| 按身份浏览用户档案 | 在UI [!DNL Platform] 中，您现在可以选择标识命名空间并提供标识值以浏览用户档案。 |

## 源 {#sources}

Adobe Experience Platform可以从外部源收集数据，同时允许您使用服务构建、标记和增强该 [!DNL Platform] 数据。 您可以从各种来源(如Adobe应用程序、基于云的存储、第三方软件和您的CRM系统)收集数据。

[!DNL Experience Platform] 提供REST风格的API和交互式UI，让您可以轻松为各种数据提供者设置源连接。 这些源连接允许您验证并连接到外部存储系统和CRM服务，设置获取运行的时间，以及管理数据获取吞吐量。

**新增功能**

| 功能 | 描述 |
| ------- | ----------- |
| 对云存储系统的更多API和UI支持 | 新的源连接器 [!DNL Azure File Storage]。 |
| 对数据库的其他API和UI支持 | IBM DB2和 [!DNL Azure Data Explorer]IBM DB2的新源连接器 [!DNL Oracle DB]。 |
| Adobe Audience Manager到 [!DNL Experience Platform] 数据共享 | 连接器的设置 [!DNL Audience Manager] 过程已更新。 [!DNL Audience Manager] 现在， [!DNL Real-time Customer Profile] 默认情况下禁用的数据集。 您可以手动选择要提升到的数据集 [!DNL Profile]。 新的默认设置不具有追溯性，并且仅影响新连接器的 [!DNL Audience Manager] 设置。 请参阅数据集用户指 [南中的更多信息](../../catalog/datasets/user-guide.md)。 |
