# Adobe Experience Platform 发行说明

**发行日期：2020 年 5 月 13 日**

Adobe Experience Platform 现有功能的更新包括：

- [用户界面更新](#ux)
- [[!DNL Data Science Workspace]](#dsw)
- [[!DNL Destinations]](#destinations)
- [Experience Platform Web SDK和Experience Platform Edge Network](#edge)
- [[!DNL Real-time Customer Profile]](#profile)
- [[!DNL Sources]](#sources)

## 用户界面更新 {#ux}

Adobe Experience Platform将发布对域和标题栏的更新，以改进您的体验并与其他[!DNL Experience Cloud]应用程序统一。

- 更轻松地在组织之间切换或切换到其他应用程序
- 改进了用户帮助，包括“帮助”菜单中的特色文章和与上下文相关的文档
- 能提供有关[!DNL Experience Platform]和文件支持票证的反馈

新体验的推出是渐进的。 您可以在[https://experience.adobe.com/platform](https://experience.adobe.com/platform)视图体验。

## [!DNL Data Science Workspace] {#dsw}

[!DNL Data Science Workspace] 使用机器学习和人工智能从数据中释放洞察。[!DNL Data Science Workspace]集成到Adobe Experience Platform中，可帮助您跨Adobe解决方案使用内容和数据资产进行预测。 [!DNL Data Science Workspace]实现此操作的方法之一是使用[!DNL JupyterLab]。 [!DNL JupyterLab] 是一个基于Web的用户界面， [[!DNL Project Jupyter]](https://jupyter.org/) 它与Adobe Experience Platform紧密集成。它为数据科学家提供了一个交互式开发环境，以便他们能够使用[!DNL Jupyter]笔记本、代码和数据。

**新增功能**

| 功能 | 描述 |
|--- | ---|
| [!DNL JupyterLab Launcher] | [!DNL JupyterLab Launcher]现在包括[!DNL Spark] 2.4笔记本的入门。 [!DNL Spark] 2.3笔记本电脑入门程序现在标记为已弃用，并在后续版本中设置为删除。 |
| [!DNL Spark] 2.4 | 新的Scala([!DNL Spark])和PySpark方法现在使用[!DNL Spark] 2.4。 |
| 内核 | 现在，通过Scala内核创作了Scala([!DNL Spark])笔记本。 PySpark笔记本现在通过[!DNL Python]内核进行创作。 [!DNL Spark]和PySpark内核已弃用，并设置为在后续发行版中删除。 |
| 菜谱 | 新的PySpark和[!DNL Spark]菜谱现在遵循与[!DNL Python]和R菜谱类似的[!DNL Docker]工作流。 |

有关迁移笔记本电脑和方法以使用[!DNL Spark] 2.4的详细信息，请参阅[笔记本数据访问指南](../../data-science-workspace/jupyterlab/access-notebook-data.md)。 有关数据科学工作区的更多一般信息，请参阅[概述文档](../../data-science-workspace/home.md)。

## [!DNL Experience Platform Web SDK] 和 [!DNL Experience Platform Edge Network] {#edge}

[!DNL Experience Platform Web SDK]和[!DNL Experience Platform Edge Network]允许用户将数据实时发送到Adobe Experience Platform和其他Adobe解决方案，以供最终用户设备和浏览器使用。 最近的使用案例列表可在我们的[公共路线图](https://github.com/adobe/alloy/projects/5)中找到，该路线图经常更新。

**新增功能**

| 功能 | 描述 |
|--- | ---|
| 支持ECID | SDK支持开箱即用的ECID，无需安装任何其他库或信息。 |
| 配置UI | 使用[!DNL Launch]中新的边缘配置UI管理配置ID设置，您必须位于允许列表才能访问。 |
| Adobe Experience Platform Web SDK模式字段组 | 用于[!DNL Experience Platform web SDK]的字段组，包含所有受支持的字段。 |
| 课程同意控制 | 为公司提供对[!DNL Experience Platform Web SDK]的加入和退出的控制。 |
| 新[!DNL Experience Cloud Debugger Extension]中的客户端调试支持 | 查看来自[!DNL Experience Platform web SDK]和边缘跟踪的请求，了解数据如何通过系统流动。 |
| Adobe Analytics | 通过边缘配置将数据发送到[!DNL Analytics]报表包。 XDM被拼合到上下文数据中，支持多包标记。 |
| Adobe Target | 支持Adobe Target。 包括VEC、基于表单的书写器、A/B、XT、Automated Personalization、MVT。 |
| Adobe Audience Manager支持 | 支持[!DNL Audience Manager] ID同步、URL目标和Cookie目标。 |
| 身份同步 | 将`setCustomersIds`重命名为`syncIdentity`以更加清晰。 |
| XDM Object Builder | 在启动扩展中，您现在可以将XDM对象构建为数据元素。 |

有关[!DNL Platform Web SDK]和[!DNL Edge Network]的详细信息，请参阅[文档](../../edge/home.md)。

## [!DNL Real-time Customer Profile] {#profile}

Adobe Experience Platform使您能够在客户与您的品牌互动的任何地方或何时为其提供协调一致的相关体验。 通过[!DNL Real-time Customer Profile]，您可以看到每个客户的整体视图，该客户综合了来自多个渠道的数据，包括在线、离线、CRM和第三方数据。 [!DNL Profile] 允许您将不同的客户数据整合到一个统一的视图中，为每次客户互动提供一个可操作、有时间戳的帐户。

**新增功能**

| 功能 | 描述 |
| -----------| ---------- |
| 新的用户档案导出量度 | 为用户档案导出作业添加了量度，显示每个命名空间中导出的用户档案总数和用户档案数。 |
| 新的可观察性洞察指标 | 可观察性洞察API现在具有以下指标，用于向[!DNL Profile]中流化接收：传入请求率、成功摄取率、摄取的记录大小。 |
| 批量GET端点 | 向[!DNL Real-time Customer Profile] API添加了批量GET端点，以在单个API调用中检索多个结果。 现在，您最多可以对区段定义、区段作业和合并策略的100个ID进行GET。 |
| 按身份浏览用户档案 | 在[!DNL Platform] UI中，您现在可以选择标识命名空间并提供标识值以浏览用户档案。 |

## 源 {#sources}

Adobe Experience Platform可以从外部源收集数据，同时允许您使用[!DNL Platform]服务来构建、标记和增强该数据。 您可以从各种来源收集数据，如Adobe应用程序、基于云的存储、第三方软件和您的CRM系统。

[!DNL Experience Platform] 提供RESTful API和交互式UI，让您可以轻松为各种数据提供者设置源连接。这些源连接允许您对外部存储系统和CRM服务进行身份验证并连接，设置获取运行的时间，以及管理数据获取吞吐量。

**新增功能**

| 功能 | 描述 |
| ------- | ----------- |
| 云存储系统的其他API和UI支持 | [!DNL Azure File Storage]的新源连接器。 |
| 对数据库的其他API和UI支持 | [!DNL Azure Data Explorer]、IBM DB2和[!DNL Oracle DB]的新源连接器。 |
| Adobe Audience Manager到[!DNL Experience Platform]数据共享 | [!DNL Audience Manager]连接器的设置过程已更新。 [!DNL Audience Manager] 现在， [!DNL Real-time Customer Profile] 默认情况下禁用的数据集。您可以手动选择要提升到[!DNL Profile]的数据集。 新的默认设置不具有追溯性，并且仅影响新[!DNL Audience Manager]连接器的设置。 请参阅[数据集用户指南](../../catalog/datasets/user-guide.md)中的详细信息。 |

**已知问题**

- 没有。
