## [!DNL Data Ingestion] {#ingestion}

Adobe Experience Platform提供了丰富的功能集，用于摄取任何类型的数据和延迟。 您可以使用批量API或流API，通过Adobe构建的连接器、数据集成合作伙伴或Adobe Experience Platform UI进行摄取。

有关产品文档，请参阅 [数据摄取概述](../../ingestion/home.md).

### 批量摄取

使用批量摄取API将任何类型的数据摄取到Adobe Experience Platform。 批量摄取API提供了一种安全且可扩展的机制，用于摄取符合XDM或非XDM模式的数据。

| 功能 | 描述 |
| -----------| ---------- |
| 富格式支持 | 支持摄取JSON、Parquet、CSV |
| 数据验证 | 对数据执行三个级别的验证：架构、数据类型和约束 |
| 移动中的数据加密 | 在数据跨数据网络传输时保护和保护数据。 |
| 分段上传 | 文件256MB或更大时支持的大文件上传 |
| 自动分区 | 数据会自动进行分区，以提高可扩展性并优化大数据集的性能。 |

**已知问题**

None.

有关产品文档，请参阅 [批量摄取概述](../../ingestion/batch-ingestion/overview.md).

### 连接器

Adobe Experience Platform连接器可帮助您从多个源轻松摄取数据，从而允许您使用来构建、标记和增强数据 [!DNL Experience Platform] 服务。 [!DNL Platform] 连接器允许您对基于云的存储系统和CRM服务进行身份验证，设置摄取运行的时间，以及管理数据摄取吞吐量。

| 功能 | 描述 |
| -----------| ---------- |
| 基于云的存储连接器 | 从中摄取数据 [!DNL Microsoft Azure Blob] 和 [!DNL Amazon S3] 云存储到Adobe Experience Platform |
| [!DNL Microsoft Dynamics CRM] &amp; [!DNL Salesforce CRM] | 发现 [!DNL Microsoft Dynamics] 或 [!DNL Salesforce] CRM。 从中摄取任何对象、标准对象或自定义对象 [!DNL Microsoft Dynamics] 或 [!DNL Salesforce] CRM。 增量数据可以每小时或每天摄取一次。 |  |

**已知问题**

* 基于云的存储连接器只能通过API调用使用。


有关产品文档，请参阅 [连接器概述](../../sources/home.md).

### Adobe解决方案

Adobe提供从Adobe解决方案到Adobe Experience Platform的托管数据传输。 在此版本中，可从以下解决方案中传输数据。

| 功能 | 描述 |
| -----------| ---------- |
| Adobe Analytics | 来自Adobe Analytics的报表包数据可在 [!DNL Platform]. |
| Adobe Audience Manager | 从Adobe Audience Manager收集数据日志、载入数据、特征和区段数据可在 [!DNL Platform]. |
| [!DNL Advertising Cloud] | DSP日志来自 [!DNL Advertising Cloud] 在 [!DNL Platform]. |
| Adobe Campaign Standard | 来自Adobe Campaign Standard的用户档案和事件数据可在 [!DNL Platform]. |
| [!DNL Target] (预览) | 来自Adobe Target的事件数据可在 [!DNL Platform]. |


**已知问题**

无。

### 流式摄取

流摄取将数据发送到 [!DNL Experience Platform] 实时从客户端和服务器端设备访问。 可以对 [!DNL Real-Time Customer Profiles] 在亚秒延迟，支持经过验证的数据收集和多记录负载。

| 功能 | 描述 |
| -----------| ---------- |
| 同步验证 | 允许立即发现JSON有效负载中的错误，从而提高开发人员速度。 |
| 端到端监控 | 监控从摄取到消费的流数据 [!DNL Data Lake] 和 [!DNL Real-time Customer Profile]. |
| 已验证的数据收集 | 打开或关闭需要客户端验证令牌的选项。 允许Adobe Experience Platform服务区分来自可信来源和不可信来源的记录。 |
| 通过实时发送数据 [!DNL Launch] | 在中使用Adobe Experience Platform扩展 [!DNL Launch] 以便从客户端设备实时轻松发送XDM格式的数据。 |

**已知问题**

无。

有关产品文档，请参阅 [流摄取概述](../../ingestion/streaming-ingestion/overview.md).

### 监控

实时监控从客户端和服务器端设备向Adobe Experience Platform的批量摄取和流式引入。

| 功能 | 描述 |
| -----------| ---------- |
| 监控流数据 | 端到端监控流式数据到 [!DNL Data Lake], [!DNL Real-Time Customer Profile]和 [!DNL Identity Service]. |

## [!DNL Experience Data Model] (XDM) {#xdm}

标准化和互操作性是其背后的关键概念 [!DNL Experience Platform]. [!DNL Experience Data Model] (XDM)在Adobe的驱动下，努力标准化客户体验数据并定义客户体验管理架构。

XDM是一项公开记录的规范，旨在提高数字体验的强大功能。 它为任何与Adobe Experience Platform上的服务通信的应用程序提供了通用结构和定义。 通过遵循XDM标准，所有客户体验数据都可以整合到通用呈现形式中，以更快、更集成的方式提供洞察。 您可以从客户操作中获得有价值的分析，通过区段定义客户受众，以及将客户属性用于个性化目的。

XDM是允许 [!DNL Experience Clou]d由Adobe Experience Platform提供支持，在适当的时间，在适当的渠道，向适当的人员传达正确的信息。

方法 [!DNL Experience Platform] 是建的， [!DNL XDM System] 操作 [!DNL Experience Data Model] 供使用的架构 [!DNL Experience Platform] 组件。

| 功能 | 描述 |
| -----------| ---------- |
| [!DNL Schema Registry] | 的 [!DNL Schema Registry] 提供用户界面([!DNL Schema Editor])和RESTful API中，可从中查看和管理所有与架构相关的资源。 的 [!DNL Schema Editor] 和 [!DNL Schema Registry] API还可用于创建和管理每个IMS组织特有的新架构和资源。 |
| [!DNL Schema Library] | 的 [!DNL Schema Library] 包含全部 [!DNL Schema Registry] 资源，包括由单个IMS组织定义的资源和按Adobe提供给所有用户的资源， [!DNL Experience Platform] 与其应用程序集成的合作伙伴和供应商 [!DNL Experience Platform]. |

**已知问题**

无。

要了解有关使用XDM的更多信息，请使用 [!DNL Schema Registry] API和 [!DNL Schema Editor]，请阅读 [XDM系统文档](../../xdm/home.md).

## [!DNL Real-time Customer Profile] {#profile}

[!DNL Profile] 是一个通用的查找实体存储，用于合并各种企业数据资产中的数据，并以实时用户档案和相关时间系列事件（称为并集视图）的形式提供对该数据的访问。 统一的视图有助于在各个渠道中为操作提供信息并赋予其权能， [!DNL Platform]和Adobe解决方案集成。 此外， [!DNL Profile] 用于支持机器学习和Sensei。 [!DNL Profile] API还可直接用于扩充第三方解决方案、CRM系统和专有解决方案的功能。

| 功能 | 描述 |
| -----------| ---------- |
| [!DNL Real-time customer profile] | [!DNL Profile] 通过数据入口摄取数据，将其与配置文件存储中的现有数据合并，并近乎实时地创建或修改统一视图。 此数据会随着客户体验的不断变化，可用于提供个性化推荐。 |
| 并集视图 | 的主要功能之一 [!DNL Profile] 是统一多渠道数据的功能。 [!DNL Profile] 可以为客户群中的各个人员、跨数据集提供合并视图，并为链接的身份（称为合并视图）提供合并视图。 |
| Edge | 边缘服务可作为在所有可寻址渠道中进行低延迟数据收集、可插拔计算和快速数据激活的框架。 边缘服务有助于实时个性化。 |

**已知问题**

无。

有关产品文档，请参阅 [用户档案概述](../../profile/home.md).

## 身份服务 {#identity}

Identity Service可解决客户身份不连带来的根本难题，这是由于客户数据在贵组织生态系统中各个不同系统中分散的性质所致。 它通过跨设备和跨消费者与您的品牌互动所使用的各种系统来桥接身份来实现这一点。 这样，Identity Service便于您全面了解客户及其行为，以便您能够实时提供有影响的数字体验。

| 功能 | 描述 |
| -----------| ---------- |
| 身份图 | 身份图是身份之间关系的映射，可近乎实时地随客户活动进行更新。 [!DNL Identity Service] 管理仅由您的组织可见且基于您的数据（称为专用图）构建的身份图。 [!DNL Identity Service] 当摄取的数据记录包含多个身份时，通过添加这些身份之间的关系来增强您的专用图。 |

**已知问题**

无。

有关产品文档，请参阅 [Identity Service概述](../../identity-service/home.md).

## [!DNL Segmentation Service] {#segmentation}

[!DNL Segmentation Service] 从用户档案存储中定义特定的用户档案子集，用于描述区分用户档案存储中可销售人群的标准。 区段可以基于记录数据（如人口统计信息）或代表客户与您品牌接触点的时间系列事件。

例如，在名为“您忘记购买运动鞋吗？”的电子邮件促销活动中，您可能希望获得一个受众区段，该区段包含过去30天内搜索跑鞋但未完成购买的所有用户。 或者，您可能希望将Web内容定位为仅向在奖励计划中具有特定评级的用户显示。

| 功能 | 描述 |
| -----------| ---------- |
| 分段API | RESTful服务使您能够以编程方式访问构建客户区段。 |
| 区段生成器 | 区段生成器是一个功能强大且直观的工作区，可用于从您的 [!DNL Profile] 数据。 |

**已知问题**

无。

有关产品文档，请参阅 [Segmentation Service概述](../../segmentation/home.md)

## 数据管理 {#governance}

Adobe Experience Platform允许公司将多个企业系统中的数据整合在一起，以便营销人员更好地识别、了解和吸引客户。 [!DNL Experience Platform] 包括端到端数据管理基础架构，以确保在 [!DNL Platform] 和在系统之间共享时。

数据管理是用于管理客户数据并确保遵守适用于数据使用的法规、限制和策略的一系列策略和技术。 它在 [!DNL Experience Platform] 在不同级别，包括编目、数据谱系、数据使用标签、数据访问策略以及对营销操作数据的访问控制。

要开始进行数据管理，必须全面了解适用于您的客户数据的法规、合同义务和公司政策。 在此，数据可以通过应用适当的数据使用标签来分类，并且其使用可以通过定义数据使用策略来控制。

数据管理框架通过 [!DNL Experience Platform] 用户界面和 [!DNL Policy Service] API。

| 功能 | 描述 |
| -----------| ---------- |
| 标签 [!DNL Experience Platform] 实体 | 根据预定义的分类标签在连接、数据集和数据集字段级别应用和管理数据使用情况标签。 |
| 创建数据使用策略 | 根据标签和请求的营销操作的存在定义和管理数据使用策略。 |
| 检查策略违规 | 评估对执行营销操作时是否违反了定义的使用策略 [!DNL Platform] 数据。 |

**已知问题**

* 无。

有关详细信息，请参阅 [数据管理概述](../../data-governance/home.md))。
