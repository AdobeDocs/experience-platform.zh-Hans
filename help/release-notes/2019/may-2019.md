## [!DNL Data Ingestion] {#ingestion}

Adobe Experience Platform提供丰富的功能集，可采集任何类型和任何延迟的数据。 您可以使用批处理或流式API，使用Adobe构建的连接器、数据集成合作伙伴或Adobe Experience PlatformUI进行采集。

有关产品文档，请参阅 [数据摄取概述](../../ingestion/home.md)。

### 批量摄取

使用Batch Ingestion API将任何类型的数据引入Adobe Experience Platform。 Bulk Ingestion API提供了安全、可伸缩的机制，用于摄取符合XDM或非XDM模式的数据。

| 功能 | 描述 |
| -----------| ---------- |
| 丰富格式支持 | 支持获取JSON、Parke、CSV |
| 数据验证 | 对数据执行了三个级别的验证：模式、数据类型和约束 |
| 动态数据加密 | 在跨数据网络传输数据时保护和保护数据。 |
| 多部件上传 | 文件支持256MB或更大的大文件上传 |
| 自动分区 | 数据会自动进行分区，以提高可伸缩性并优化大数据集的性能。 |

**已知问题**

None.

有关产品文档，请参阅批 [处理摄取概述](../../ingestion/batch-ingestion/overview.md)。

## [!DNL Experience Data Model] (XDM) {#xdm}

标准化和互操作性是背后的关键概念 [!DNL Experience Platform]。 [!DNL Experience Data Model] (XDM)由Adobe驱动，旨在实现客户体验数据标准化并定义客户体验管理模式。

XDM是一个公开记录的规范，旨在提高数字体验的强大功能。 它为任何与Adobe Experience Platform上的服务通信的应用程序提供了通用结构和定义。 通过遵守XDM标准，所有客户体验数据都可以整合到一个通用表现形式中，以更快、更集成的方式提供洞察。 您可以从客户行动中获得宝贵的洞察，通过细分定义客户受众，并将客户属性用于个性化目的。

XDM是一种机制，它 [!DNL Experience Clou]允许由Adobe Experience Platform提供动力的d在正确的渠道，在恰当的时刻，向正确的人提供正确的信息。

所构建的方法 [!DNL Experience Platform] 可操作 [!DNL XDM System] 模式, [!DNL Experience Data Model] 供组件使 [!DNL Experience Platform] 用。

| 功能 | 描述 |
| -----------| ---------- |
| [!DNL Schema Registry] | 它提 [!DNL Schema Registry] 供了用户界面([!DNL Schema Editor])和RESTful API，从中可以查看和管理所有与模式相关的资源。 该 [!DNL Schema Editor] 和 [!DNL Schema Registry] API还可用于创建和管理每个IMS组织特有的新模式和资源。 |
| [!DNL Schema Library] | 它包 [!DNL Schema Library] 含所 [!DNL Schema Registry] 有资源，包括由单个IMS组织定义的资源，以及由Adobe、合作伙伴和与其应用程序集成的供应商向所有用户提供的资 [!DNL Experience Platform][!DNL Experience Platform]源。 |

**已知问题**

None.

要进一步了解如何使用API [!DNL Schema Registry] 和XDM, [!DNL Schema Editor]请阅读 [XDM系统文档](../../xdm/home.md)。

## [!DNL Real-time Customer Profile] {#profile}

[!DNL Profile] 是通用的查找实体存储，它合并各种企业用户档案资产中的数据，并以实时消费者和相关时间序列事件的形式提供对该数据的访问，称为合并视图。 统一的视图有助于在渠道、和Adobe解决方案集成中 [!DNL Platform]通知并支持各种操作。 此外， [!DNL Profile] 还用于支持机器学习和Sensei。 [!DNL Profile] API还可以直接用于丰富第三方解决方案、CRM系统和专有解决方案的功能。

| 功能 | 描述 |
| -----------| ---------- |
| [!DNL Real-time customer profile] | [!DNL Profile] 通过用户档案入口接收数据，将其与视图存储中的现有数据合并，并几乎实时创建或修改统一数据。 这些随着客户体验的不断发展而不断变化的数据可以用于提供个性化的推荐。 |
| 合并视图 | 其中一个关键特 [!DNL Profile] 征是能够统一多渠道数据。 [!DNL Profile] 可以为消费者群中的个人提供合并视图、跨数据集以及链接身份(称为合并视图)。 |
| Edge | Edge Service作为低延迟数据收集、可插拔计算和跨所有可寻址渠道快速激活的框架。 Edge Service可促进实时个性化。 |

## Identity Service {#identity}

Identity Service解决了客户身份分离所带来的根本挑战，这是由于客户数据在组织的生态系统中的每个不同系统中分散而造成的。 它通过跨设备和跨消费者与品牌互动所使用的各种系统连接身份来实现这一点。 通过这种方式，Identity Service可帮助您全面了解客户及其行为，从而实时提供有影响力的数字体验。

| 功能 | 描述 |
| -----------| ---------- |
| 身份图 | 标识图是标识之间关系的映射，可以随客户活动实时更新。 [!DNL Identity Service] 管理仅由您的组织可见的标识图，并根据您的数据（称为专用图）构建。 [!DNL Identity Service] 当收录的数据记录包含多个身份时增加您的私人图表，并在这些身份之间添加关系。 |

**已知问题**

None.

有关产品文档，请参 [阅Identity Service概述](../../identity-service/home.md)。

## [!DNL Segmentation Service] {#segmentation}

[!DNL Segmentation Service] 定义用户档案商店中特定的用户档案子集，描述区分用户档案商店中有价人群的标准。 细分可以基于记录事件（如人口统计信息）或代表客户与您品牌接触点的时间序列数据。

例如，在名为“您忘记购买运动鞋吗？”的电子邮件活动中，您可能希望在过去30天内搜索跑步鞋但未完成购买的所有用户的受众区段。 或者，您可能希望目标Web内容，以便仅向奖励项目中具有特定评级的用户显示。

| 功能 | 描述 |
| -----------| ---------- |
| 分段API | RESTful服务使您能够以程序化方式建立客户细分。 |
| 区段生成器 | 细分构建器是一个功能强大且直观的工作区，可根据您的数据构建客户 [!DNL Profile] 细分。 |

**已知问题**

None.

有关产品文档，请参阅 [Segmentation Service概述](../../segmentation/home.md)

## [!DNL Data Governance] {#governance}

Adobe Experience Platform允许公司将来自多个企业系统的数据整合在一起，从而更好地使营销人员能够识别、理解和吸引客户。 [!DNL Experience Platform] 包括端对端数据治理基础架构，确保在系统内和系统之间 [!DNL Platform] 正确使用数据。

数据治理是用于管理客户数据并确保遵守适用于数据使用的法规、限制和政策的一系列战略和技术。 它在各个层次中都起 [!DNL Experience Platform] 着关键作用，包括编目、数据谱系、数据使用标签、数据访问策略和访问控制营销行动的数据。

要开始进行数据治理，必须全面了解适用于客户数据的法规、合同义务和公司政策。 从那里，可以通过应用适当的数据使用标签来对数据进行分类，并且可以通过定义数据使用策略来控制其使用。

该框 [!DNL Data Governance] 架通过用户界面和API简化数据分类和创建数据使用策略 [!DNL Experience Platform] 的过程并简 [!DNL Policy Service] 化。

| 功能 | 描述 |
| -----------| ---------- |
| 标签实 [!DNL Experience Platform] 体 | 根据预定义的分类标签在连接、数据集和数据集字段级别应用和管理数据使用标签。 |
| 创建数据使用策略 | 根据标签和请求的营销操作来定义和管理数据使用策略。 |
| 检查违反策略的情况 | 评估在对数据执行营销操作时是否违反了定义的使用 [!DNL Platform] 策略。 |

**已知问题**

* None.

有关详细信息，请参阅 [数据管理概述](../../data-governance/home.md))。
