## 数据摄取

Adobe Experience Platform提供一组丰富的功能，用于摄取任何类型和任何延迟的数据。 您可以使用Adobe构建的连接器、数据集成合作伙伴或Adobe Experience Platform UI使用批处理或流API进行摄取。

有关产品文档，请参阅数 [据摄取概述](../../ingestion/home.md)。

### 批量摄取

使用Batch Ingestion API将任何类型的数据摄取到Adobe Experience Platform中。 Bulk Ingestion API提供了安全、可伸缩的机制，用于摄取符合XDM或非XDM模式的数据。

| 功能 | 描述 |
| -----------| ---------- |
| 富格式支持 | 支持摄取JSON、Parke、CSV |
| 数据验证 | 对数据执行三个级别的验证：模式、数据类型和约束 |
| 动态数据加密 | 在跨数据网络传输数据时保护并保护数据。 |
| 多部分上传 | 文件支持256MB或更大的大文件上传 |
| 自动分区 | 数据自动分区以提高可伸缩性并优化大数据集的性能。 |

**已知问题**

无.

有关产品文档，请参阅批 [量摄取概述](../../ingestion/batch-ingestion/overview.md)。

### 连接器

Adobe Experience Platform Connectors可帮助您轻松从多个来源获取数据，从而使您能够使用Experience Platform服务来构建、标记和增强数据。 平台连接器允许您验证到基于云的存储系统和CRM服务，为摄取运行设置时间并管理数据摄取吞吐量。

| 功能 | 描述 |
| -----------| ---------- |
| 基于云的存储连接器 | 将数据从Microsoft Azure Blob和Amazon S3云存储收录到Adobe Experience Platform |
| Microsoft Dynamics CRM和Salesforce CRM | 发现Microsoft Dynamics或Salesforce CRM中可用的所有数据。 从Microsoft Dynamics或Salesforce CRM中摄取任何对象、标准或自定义。 增量数据可以以每小时或每天的速度摄取。 |  |

**已知问题**

有关产品文档，请参阅 [连接器概述](../../source-connectors/home.md)。

| 功能 | 描述 |

### 流摄取

流摄取从客户端和服务器端设备实时将数据发送到Experience Platform。 可在次秒延迟时对实时客户用户档案进行更新，支持实名数据收集和多记录负载。

| 功能 | 描述 |
| -----------| ---------- |
| 同步验证 | 允许立即发现JSON有效负荷中的错误，从而提高开发人员速度。 |
| 端到端监控 | 在数据库和实时客户用户档案中监控从摄取到消费的流数据。 |
| 已验证的数据收集 | 打开或关闭选项以要求客户端提供身份验证令牌。 允许Adobe Experience Platform服务区分来自可信来源和不可信来源的记录。 |
| 通过Launch实时发送数据 | 使用Launch中的Adobe Experience Platform扩展，从客户端设备轻松实时发送XDM格式的数据。 |

无.

## 体验数据模型(XDM)和XDM系统

标准化和互操作性是Experience Platform的主要概念。 Adobe推动的体验数据模型(XDM)旨在标准化客户体验数据并定义客户体验管理模式。

XDM是一个公开的规范，旨在提高数字体验的强大功能。 它为与Adobe Experience Platform上的服务通信的任何应用程序提供了通用结构和定义。 通过遵守XDM标准，所有客户体验数据都可以整合到以更快、更集成的方式提供洞察的共同表现形式中。 您可以从客户行动中获得宝贵的洞察，通过细分定义客户受众，并将客户属性用于个性化目的。

XDM是一种机制，它允许Experience Cloud以Adobe Experience Platform为后盾，在正确的时间在正确的渠道上向正确的人员提供正确的信息。

XDM系统构建Experience Platform的方法，可操作Experience Data Model模式，供Experience Platform组件使用。

| 功能 | 描述 |
| -----------| ---------- |
| 模式注册 | 模式注册表提供用户界面(模式编辑器)和RESTful API，从中可以查看和管理所有与模式相关的资源。 模式编辑器和模式注册表API还可用于创建和管理每个IMS组织特有的新模式和资源。 |
| 模式库 | 模式库包含所有模式注册表资源，包括由IMS个人组织定义的资源，以及Adobe、Experience Platform合作伙伴和与Experience Platform集成的应用程序的供应商向所有用户提供的资源。 |

**已知问题**

无.

要了解有关使用模式注册表API和模式编辑器使用XDM的更多信息，请阅读 [XDM系统文档](../../xdm/home.md)。

### 配置文件

用户档案是一个通用查找实体存储，它合并各种企业数据资产中的数据，并提供以实时消费者用户档案和相关时间序列事件(称为合并视图)形式访问该数据。 统一的视图有助于通知和支持跨渠道、平台和Adobe解决方案集成的操作。 此外，用户档案还用于支持机器学习和Sensei。 用户档案API还可以直接用于丰富第三方解决方案、CRM系统和专有解决方案的功能。

| 功能 | 描述 |
| -----------| ---------- |
| 实时客户用户档案 | 用户档案通过数据入口获取数据，将其与用户档案存储中的现有数据合并，并几乎实时创建或修改统一视图。 这些随着客户体验的不断发展而不断变化的数据可用于提供个性化的推荐。 |
| 合并视图 | 用户档案的关键特征之一是统一多渠道数据的能力。 用户档案可以为消费者群中的个人、跨数据集和链接标识(称为合并视图)提供合并视图。 |
| Edge | Edge Service作为一个框架，用于在所有可寻址渠道中低延迟数据收集、可插拔计算和快速数据激活。 Edge Service可促进实时个性化。 |

**已知问题**

无.

有关产品文档，请参阅 [用户档案概述](../../profile/home.md)。

### 标识服务

Identity Service解决了客户身份不连贯所带来的根本挑战，这是由于客户数据在组织的生态系统中处于不同运行状态的各个系统中分散的特性造成的。 它通过跨设备和跨消费者使用的、用于与您的品牌互动的各种系统连接身份来实现这一点。 这样，Identity Service便于您全面了解客户及其行为，从而使您能够实时提供有影响力的数字体验。

| 功能 | 描述 |
| -----------| ---------- |
| 标识图 | 标识图是标识之间关系的映射，可以随客户活动实时更新。 Identity Service管理仅由您的组织可见的标识图，并基于您的数据（称为专用图）构建。 当所摄取数据的记录包含多个身份时，Identity Service会增强您的私人图形，从而添加这些身份之间的关系。 |

**已知问题**

无.

有关产品文档，请参 [阅Identity Service概述](../../identity-service/home.md)。

### 分段服务

细分服务从用户档案商店定义特定的用户档案子集，描述区分用户档案商店中有价人群的标准。 细分可以基于记录数据（如人口统计信息）或表示客户与您品牌接触点的时间序列事件。

例如，在名为“您忘记购买运动鞋吗？”的电子邮件活动中，您可能希望获得在过去30天内搜索跑鞋但未完成购买的所有用户的受众区段。 或者，您可能希望将Web内容目标为仅向奖励项目中具有特定评级的用户显示。

| 功能 | 描述 |
| -----------| ---------- |
| 分段API | RESTful服务使您能够以程序化方式建立客户细分。 |
| 区段生成器 | 细分构建器是一个强大而直观的工作区，可根据您的用户档案数据构建客户细分。 |

**已知问题**

## 数据管理

Adobe Experience Platform允许公司将来自多个企业系统的数据整合在一起，从而更好地允许营销人员识别、理解和吸引客户。 Experience Platform包括端对端数据管理基础架构，包括数据使用标签和强制执行(DULE)，以确保在平台内以及在系统之间共享数据时正确使用数据。

数据管理是用于管理客户数据并确保遵守适用于数据使用的法规、限制和政策的一系列战略和技术。 它在Experience Platform的不同级别中起着关键作用，包括编目、数据谱系、数据使用标签、数据访问策略和营销操作的数据访问控制。

要开始进行数据管理，您必须全面了解适用于客户数据的法规、合同义务和公司政策。 从那里，可以通过应用适当的数据使用标签来对数据进行分类，并且可以通过定义数据使用策略来控制其使用。

DULE框架通过Experience Platform用户界面和DULE Policy Service API简化了数据分类和创建数据使用策略的过程。

| 功能 | 描述 |
| -----------| ---------- |
| 标签体验平台实体 | 在连接、数据集和数据集字段级别基于预定义的分类标签应用和管理数据使用标签。 |
| 创建数据使用策略 | 根据标签和请求的营销操作来定义和管理数据使用策略。 |
| 检查违反策略的情况 | 评估在平台数据上执行营销操作时是否违反了定义的使用策略。 |

**已知问题**

* 无.

有关详细信息，请参阅数 [据管理概述](../../data-governance/home.md))。
