## [!DNL Data Ingestion] {#ingestion}

Adobe Experience Platform 提供了一组丰富的功能，以摄取任何类型和任何延迟的数据。您可以使用Adobe构建的连接器、数据集成合作伙伴或Adobe Experience Platform UI，使用批处理或流式处理API进行摄取。

有关产品文档，请参阅 [数据引入概述](../../ingestion/home.md).

### 批量摄取

使用批量摄取API将任何类型的数据摄取到Adobe Experience Platform中。 批量摄取API提供了一种安全且可伸缩的机制来摄取符合XDM或非XDM架构的数据。

| 功能 | 描述 |
| -----------| ---------- |
| 支持丰富格式 | 支持JSON、Parquet、CSV的引入 |
| 数据验证 | 对数据执行三个级别的验证：架构、数据类型和约束 |
| 动态数据加密 | 在跨数据网络传输数据时保护数据并保护其安全。 |
| 多部分上传 | 256MB或更大的文件支持大文件上传 |
| 自动分区 | 数据会自动分区，以提高可扩展性并优化大型数据集的性能。 |

**已知问题**

无。

有关产品文档，请参阅 [批量摄取概述](../../ingestion/batch-ingestion/overview.md).

### 连接器

Adobe Experience Platform Connectors可帮助您轻松地从多个源摄取数据，使您能够使用来构建、标记和增强数据 [!DNL Experience Platform] 服务。 [!DNL Platform] 连接器允许您向基于云的存储系统和CRM服务进行身份验证，设置摄取运行的时间，并管理数据摄取吞吐量。

| 功能 | 描述 |
| -----------| ---------- |
| 基于云的存储连接器 | 从以下来源引入数据 [!DNL Microsoft Azure Blob] 和 [!DNL Amazon S3] 云存储到Adobe Experience Platform |
| [!DNL Microsoft Dynamics CRM] 和 [!DNL Salesforce CRM] | 在中发现所有可用数据 [!DNL Microsoft Dynamics] 或 [!DNL Salesforce] CRM。 从中摄取任何对象（标准或自定义） [!DNL Microsoft Dynamics] 或 [!DNL Salesforce] CRM。 增量数据可以每小时或每天摄取。 | |

**已知问题**

* 基于云的存储连接器只能通过API调用使用。


有关产品文档，请参阅 [连接器概述](../../sources/home.md).

### 流式摄取

流式摄取将数据发送到 [!DNL Experience Platform] 客户端和服务器端设备的实时访问。 可以更新 [!DNL Real-Time Customer Profiles] 支持经过验证的数据收集和多记录负载，具有亚秒的延迟。

| 功能 | 描述 |
| -----------| ---------- |
| 同步验证 | 允许立即发现JSON有效负载中的错误，从而加快开发人员的工作速度。 |
| 端到端监控 | 在中监视从摄取到使用的流式数据 [!DNL Data Lake] 和 [!DNL Real-Time Customer Profile]. |
| 经过身份验证的数据收集 | 打开或关闭要求客户端提供身份验证令牌的选项。 允许Adobe Experience Platform服务区分来自可信来源和未可信来源的记录。 |
| 通过以下方式实时发送数据 [!DNL Launch] | 在中使用Adobe Experience Platform扩展 [!DNL Launch] 轻松地实时从客户端设备发送XDM格式的数据。 |

**已知问题**

无。

有关产品文档，请参阅 [流式摄取概述](../../ingestion/streaming-ingestion/overview.md).

## [!DNL Experience Data Model] (XDM) {#xdm}

标准化和互操作性是背后的关键概念 [!DNL Experience Platform]. [!DNL Experience Data Model] (XDM)由Adobe驱动，致力于标准化客户体验数据并定义用于客户体验管理的架构。

XDM是一个公开记录的规范，旨在提高数字体验的强大功能。 它为任何应用程序提供通用结构和定义，以便与Adobe Experience Platform上的服务进行通信。 通过遵守XDM标准，所有客户体验数据都可以纳入到通用表示中，从而以更快、更集成的方式提供见解。 您可以从客户行为中获得有价值的见解，通过区段定义客户受众，并使用客户属性实现个性化目的。

XDM是一种机制，它允许 [!DNL Experience Clou]d由Adobe Experience Platform提供支持，可在正确的时间通过正确的渠道向正确的人传递正确的信息。

用于以下项的方法 [!DNL Experience Platform] 已构建， [!DNL XDM System] 可操作 [!DNL Experience Data Model] 供使用的架构 [!DNL Experience Platform] 组件。

| 功能 | 描述 |
| -----------| ---------- |
| [!DNL Schema Registry] | 此 [!DNL Schema Registry] 提供用户界面([!DNL Schema Editor])和RESTful API，从中可以查看和管理所有与架构相关的资源。 此 [!DNL Schema Editor] 和 [!DNL Schema Registry] API还可用于创建和管理新架构和每个组织特有的资源。 |
| [!DNL Schema Library] | 此 [!DNL Schema Library] 包含所有 [!DNL Schema Registry] 资源，包括由单个组织定义的资源和按Adobe提供给所有用户的资源， [!DNL Experience Platform] 与其应用程序集成的合作伙伴和供应商 [!DNL Experience Platform]. |

**已知问题**

无。

要了解有关使用XDM的更多信息，请使用 [!DNL Schema Registry] API和 [!DNL Schema Editor]，请阅读 [XDM系统文档](../../xdm/home.md).

## [!DNL Real-Time Customer Profile] {#profile}

[!DNL Profile] 是一个通用查找实体存储，它合并各种企业数据资产中的数据，并以实时使用者配置文件和相关时间序列事件（称为联合视图）的形式提供对这些数据的访问。 统一视图有助于为跨渠道的操作提供信息并增强其功能， [!DNL Platform]和Adobe解决方案集成。 此外， [!DNL Profile] 用于推动机器学习和Sensei。 [!DNL Profile] API还可以直接用于扩充第三方解决方案、CRM系统和专有解决方案的功能。

| 功能 | 描述 |
| -----------| ---------- |
| [!DNL Real-time customer profile] | [!DNL Profile] 通过数据入口摄取数据，将其与配置文件存储中的现有数据合并，并近乎实时地创建或修改统一视图。 此数据会随着客户体验的发展而不断变化，可用于提供个性化推荐。 |
| 合并视图 | 的主要功能之一 [!DNL Profile] 能够统一多渠道数据。 [!DNL Profile] 可以提供客户群中个人、跨数据集和针对链接身份的合并视图，称为合并视图。 |
| Edge | 边缘服务可充当所有可寻址渠道的低延迟数据收集、可插拔计算和快速数据激活的框架。 边缘服务有助于实现实时个性化。 |

**已知问题**

无。

有关产品文档，请参阅 [配置文件概述](../../profile/home.md).

## 身份服务 {#identity}

Identity Service解决了客户身份断开带来的根本性挑战，这种身份断开是由于客户的数据在组织生态系统中的每个不同系统中呈现碎片化而造成的。 它通过在设备之间以及在客户用于与您的品牌互动的各种系统中桥接身份来实现这一点。 通过这种方式，Identity Service可帮助您全面了解客户及其行为，以便您能够实时提供有影响力的数字体验。

| 功能 | 描述 |
| -----------| ---------- |
| 身份图 | 身份图是随客户活动近乎实时更新的身份之间关系的映射。 [!DNL Identity Service] 管理仅由您的组织显示的基于您的数据的身份图，称为专用图。 [!DNL Identity Service] 当摄取的数据记录包含多个身份时，可以增强专用图，从而在这些身份之间添加关系。 |

**已知问题**

无。

有关产品文档，请参阅 [Identity服务概述](../../identity-service/home.md).

## [!DNL Segmentation Service] {#segmentation}

[!DNL Segmentation Service] 定义配置文件存储中的特定配置文件子集，描述区分配置文件存储中可营销人员组的标准。 区段可以基于代表客户与您的品牌接触点的记录数据（如人口统计信息）或时间序列事件。

例如，在名为“您是否忘记购买运动鞋？”的电子邮件促销活动中，您可能希望查看过去30天内搜索跑鞋但未完成购买的所有用户的受众区段。 或者，您可能希望将Web内容定位为仅向奖励计划中具有特定分级的用户显示。

| 功能 | 描述 |
| -----------| ---------- |
| 分段API | RESTful服务可让您以编程方式访问构建客户区段。 |
| 区段生成器 | 区段生成器是一个强大而直观的工作区，可用于从以下对象生成客户区段： [!DNL Profile] 数据。 |

**已知问题**

无。

有关产品文档，请参阅 [分段服务概述](../../segmentation/home.md)

## 数据治理 {#governance}

Adobe Experience Platform允许企业将来自多个企业系统的数据整合在一起，以便更好地让营销人员识别、理解客户并与他们互动。 [!DNL Experience Platform] 包括一个端到端的数据治理基础架构，以确保数据在 [!DNL Platform] 在系统之间共享时。

数据治理是用于管理客户数据并确保遵守适用于数据使用的法规、限制和策略的一系列策略和技术。 它在以下方面发挥着关键作用 [!DNL Experience Platform] 在各个级别，包括编目、数据谱系、数据使用标签、数据访问策略和对营销活动数据的访问控制。

开始使用数据治理需要彻底了解适用于您的客户数据的管理法规、合同义务和企业政策。 从那里，可以通过应用适当的数据使用标签对数据进行分类，并且可以通过定义数据使用策略来控制其使用。

数据管理框架通过简化分类数据和创建数据使用策略的流程 [!DNL Experience Platform] 用户界面和 [!DNL Policy Service] API。

| 功能 | 描述 |
| -----------| ---------- |
| 标签 [!DNL Experience Platform] 实体 | 根据预定义的分类标签，在连接、数据集和数据集字段级别应用和管理数据使用标签。 |
| 创建数据使用策略 | 根据标签存在和请求的营销操作定义和管理数据使用策略。 |
| 检查策略违规 | 评估在对以下对象执行营销活动时是否违反了定义的使用策略 [!DNL Platform] 数据。 |

**已知问题**

* 无。

欲知更多信息，请参见 [数据管理概述](../../data-governance/home.md))。
