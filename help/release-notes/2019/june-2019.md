# Adobe Experience Platform 发行说明

**发布日期：2019 年 6 月 28 日**

Adobe Experience Platform的新增功能：

* [[!DNL Data Science Workspace]](#dsw)
* [[!DNL Decisioning Service]](#decisioning)
* [[!DNL Query Service]](#query)

对现有功能的更新：

* [[!DNL Experience Data Model (XDM)]](#xdm)
* [[!DNL Segmentation Service]](#segmentation)

## [!DNL Data Science Workspace] {#dsw}

Adobe Experience Platform [!DNL Data Science Workspace]是[!DNL Experience Platform]中的一项完全托管服务，通过构建和操作机器学习模型，数据科学家可以跨Adobe解决方案和第三方系统无缝地从数据和内容生成洞察。 [!DNL Data Science Workspace] 与端对端数 [!DNL Platform] 据科学生命周期紧密集成并为其提供强大动力，包括探索和准备XDM数据，然后开发并实施模型以自动利用机器学习 [!DNL Real-time Customer Profile] 洞察丰富内容。

**主要功能**

| 功能 | 描述 |
| -----------| ---------- |
| 设置和计算隔离 | 为使数据科学家能够以安全方式在[!DNL Experience Platform]内执行不受信任的代码而配置所需的专用计算资源。 |
| 首次用户体验 | 包括各种机器学习框架和语言（如[!DNL Python]、R、PySpark和Scala [!DNL Spark]）的现成范例。 |
| 笔记本 | 为由[!DNL Jupyter Notebooks]提供支持的数据科学家/数据工程师定制环境，使他们能够利用库和流行的机器学习框架的精选列表准备数据、提取功能和开发ML模型。 |
| 数据探索 | 无缝访问引入与[!DNL Platform Data Access] SDK集成的[!DNL Platform]中的XDM数据。 |
| 数据可视化 | 能够在[!DNL Jupyter Notebooks]中执行SQL查询，以加速数据准备和功能工程。 |
| 特征管线 | 用于Scala/PySpark的API/SDK可部署功能工程管道，以将核心XDM数据转换为功能模式。 |
| 模型创作 | 使数据科学家能够专注于模型开发的模板和运行时，而不必实施用于访问数据和计算资源的基础结构代码。 您可以导入模型代码并将其操作，以便从[!DNL Platform]中的数据派生“洞察”。 |
| 企业模型管理 | 支持多租户数据模型以跟踪模型版本和关联的超参数配置，为合作伙伴生态系统提供基础。 |
| 模型评估 | 评估和优化[!DNL Python]、PySpark、R和Scala中的回归和分类模型。 |
| 模型部署 | 能够比较多个实验运行中的评估指标和配置，并将最佳模型发布为服务。 |
| 批评分 | 使用机器学习洞察丰富[!DNL Real-time Customer Profile]，或将它们作为数据集写回[!DNL Platform] |
| 计划 | 与[!DNL Platform]业务流程服务集成，通过API使用用户定义的计划自动化模型培训、评分和功能管道。 |

**已知问题**

* 计划和功能管道当前仅通过API可用，并且将在未来版本中添加UI。

有关详细信息，请访问[数据科学工作区概述](../../data-science-workspace/home.md)。

## [!DNL Decisioning Service] {#decisioning}

Adobe Experience Platform [!DNL Decisioning Service]提供了以编程方式智能地从一组可供给定个人的可用选项中选择“下一个最佳体验”、将其交付给任何渠道或应用程序以及执行报告和分析的功能。

预建的丰富优惠模型支持以不受渠道约束的方式进行“下一个最佳数据”决策的用例。

**主要功能**

| 功能 | 描述 |
| -----------| ---------- |
| 业务对象存储库 | 由JSON模式模型驱动的存储库允许开发人员创建、读取、更新和删除各种业务对象。 存储库提供通用、具有表现力的查询API以及模式感知搜索。 |
| 存储库容器 | 在业务对象存储库中，开发人员可以将其关注点与项目、业务或组织单位或项目生命周期阶段（例如，在开发和集成、升级或用于实时生产使用中）隔离。 这些隔离称为存储库容器。 |
| 角色和权限 | 使用[!DNL Admin Console]，组织可以创建和管理用户档案，以按类型、访问操作和容器授予对资源的目标访问权。 可以将用户添加到这些访问用户档案，并自动从这些策略计算有效访问权限。 |
| 预建的优惠对象模型 | 无需先构建数据模型，[!DNL Platform]开发人员可以利用预建的JSON模式和关系创建优惠目录、定义决策规则和约束以及组合优惠集合以进行决策。 |
| 基于用户档案和非用户档案数据的决策规则 | 与[!DNL Real-time Customer Profile]的紧密集成使开发人员能够创建利用用户档案数据的决策规则。 不仅可以使用用户档案属性做出决策，而且还可以基于用户档案的体验事件历史记录和与用户身份无关的业务实体（例如，流量条件、产品库存）做出决策。 任何[!DNL Experience Data Model](XDM)实体(其中[!DNL Schema Registry]中存在模式)都可用于决策规则。 规则是第一类实体，可以对任何决策选项和活动重用。 |
| 排名和上限 | 对满足给定用户的所有资格和其他限制的决策选项进行排名，并选择最佳选项。 可使用附加的每个用户和全局限制约束限制可用选项的曝光，从而在满足资源约束和用户疲劳的情况下实现个性化。 |
| [!DNL Decisioning] REST API | 可以使用简单的REST API调用[!DNL Decisioning Service]以获得给定个人的“下一最佳优惠”。 量度API可用于检查实时优惠建议和上限值。 |
| 将决策事件流化到[!DNL Data Lake]和[!DNL Query Service] | [!DNL Decisioning Service]自动创建数据集，以自动将所有XDM决策事件流化到[!DNL Data Lake]中。 然后，使用[!DNL Query Service]可以使用数据集进行分析和报告。 |
| 开发人员支持 | 自助选择加入(包含有关Adobe I/O的文档)，其中包含有关各种主题的教程。 |

**已知问题**

* 优惠数据模型不通过[!DNL Schema Registry]公开，因此只能以有限的方式扩展。 模型模式具有内置结构，允许附加自定义数据。 将来，您将能够扩展基本XDM模型类，以定义自己的自定义决策域。
* 您必须使用优惠管理域模型进行设置，并且用户和集成必须在此产品上下文中进行管理。

## [!DNL Query Service] {#query}

[!DNL Query Service] 提供了在Adobe Experience Platform中使用标准SQL查询数据的能力，以支持许多不同的分析和数据管理用例。它是一个无服务器工具，允许您连接[!DNL Data Lake]中的任何数据集，并将查询结果捕获为新数据集，以用于报告、[!DNL Data Science Workspace]或引入[!DNL Profile Service]。

您可以使用[!DNL Query Service]构建数据分析生态系统，创建消费者在各种交互渠道中的图景。 这些渠道可能包括：

* 销售点系统
* Web
* 移动设备
* CRM系统

**主要功能**

| 功能 | 描述 |
| -----------| ---------- |
| 查询编辑器 | 使用基于Web的工具编写、验证、测试和执行查询。 它包含一个控制台，用于提供有关查询执行的详细信息以及预览查询结果的能力。 |
| 数据集创建 | 通过标准SQL语法在[!DNL Experience Platform]上创建数据集。 |
| Adobe定义的函数 | 利用快捷键功能实现常见任务，如识别会话或设置归因。 |
| BI工具连接 | 使用常见BI工具中的PostgreSQL(Postgres)驱动程序连接到[!DNL Query Service]以创建报表和可视化。 支持的工具包括：[!DNL Tableau]、[!DNL Power BI]和[!DNL Looker]。 在“凭据”选项卡上查找身份验证信息。 |
| 数据库管理工具连接 | 将[!DNL Aqua Data Studio]或[!DNL DB Visualizer]连接到[!DNL Query Service]以实现数据探索和数据集创建功能。 [!DNL Query Service] 还支持来自R Studio的连接。在“凭据”选项卡上查找身份验证信息。 |
| 命令行查询工具 | 连接PSQL以能够从命令行运行查询。 |
| 查询日志 | 保留由[!DNL Query Service]执行的查询的历史记录，并使您能够查找以前的SQL，以便编辑、执行或在结果中创建数据集。 |
| 查询计划API | 计划查询，以通过此API重复执行。 |

**已知问题**

* [!DNL Query Editor] 显示查询结果的100行示例。要保留完整的结果集，请使用查询日志中的数据集创建功能。
* 近期发行版将添加对视图的支持以及将计划应用于查询的UI。

有关[!DNL Query Service]的详细信息，请参阅[产品文档](../../query-service/home.md)。

## [!DNL Experience Data Model] (XDM) {#xdm}

标准化和互操作性是[!DNL Experience Platform]背后的关键概念。 [!DNL Experience Data Model] (XDM)由Adobe驱动，旨在实现客户体验数据标准化并定义客户体验管理模式。

XDM是一个公开存档的规范，旨在提高数字体验的强大功能。 它为任何与Adobe Experience Platform上的服务通信的应用程序提供了通用结构和定义。 通过遵守XDM标准，所有客户体验数据都可以整合到以更快、更集成的方式提供洞察的常见表现形式中。 您可以从客户行动中获得宝贵的洞察，通过细分定义客户受众，并将客户属性用于个性化目的。

XDM是一种机制，它允许由Adobe Experience Platform提供支持的[!DNL Experience Cloud]在恰当的时间，在适当的渠道向适当的人提供适当的信息。

构建[!DNL Experience Platform]的方法，[!DNL XDM System]操作[!DNL Experience Data Model]模式以供[!DNL Experience Platform]组件使用。

**新增功能**

| 功能 | 描述 |
| ---------- | ------------ |
| JSON模式约束 | 以下数据类型现在在用户界面中具有其他选项来定义约束：`string` — 最小/最大长度、模式、默认值、格式(如[JSON模式draft-6](https://tools.ietf.org/html/draft-wright-json-schema-01)中定义)和`double` — 最小/最大默认值。 |
| 自定义 `$id` | 现在，在POST请求中创建资源时，您可以提供您自己的`$id`值。 |
| 模式注册表性能改进 | 优化了合并模式生成，并增强了模式缓存，从而大大缩短了API响应时间。 |

**错误修复**

* 将identityMap字段从上下文/用户档案移出并移入其自己的模式字段组，以使定义身份更直观。
* 已基于上下文/用户档案使用上下文/identitymap修补所有现有模式。
* 修复了未提供版本时的错误消息。
* 修复了[!DNL Schema Registry]为用户档案合并模式调用提供随机响应的错误。
* 修复了合并模式未在[!DNL Schema Registry]中显示正确字段的错误。
* 修复了标识描述符有时无法使用有效命名空间创建的错误。
* 修复了对象使用`properties`而不是`allOf`时的取消引用问题。

**已知问题**

* 无法通过添加字段来扩展[!DNL Platform]提供的字段组。
* 在从模式合成中删除字段组时，不会删除描述符。
* 无法创建没有标签的枚举字段。

要进一步了解如何使用[!DNL Schema Registry] API和[!DNL Schema Editor]使用XDM，请阅读[ XDM系统文档](../../xdm/home.md)。

## [!DNL Segmentation Service] {#segmentation}

[!DNL Segmentation Service] 定义用户档案商店中特定的用户档案子集，描述区分用户档案商店中有价人群的标准。细分可以基于记录数据（如人口统计信息）或表示客户与您品牌接触点的时间序列事件。

例如，在专注于跑鞋的电子邮件活动中，您可以使用在过去30天内搜索跑鞋但未完成购买的所有用户的受众段。 另一个示例是使用区段来目标网站内容，以便仅向属于您奖励项目某一层的访客显示。

**新增功能**

| 功能 | 描述 |
| -----------| ---------- |
| 相对时间规则 | 您现在可以选择滚动时间窗口，如14天前、3到5小时前等。 |
| XDM字段摘要 | 对于左边栏上的“属性”，现在可使用摘要，为基础数据提供视图。 |
| 左边栏搜索 | 改进了左边栏的段部分的搜索功能。 |
| eVar | 改进了对易记名称的支持，使您能更轻松地查看从Adobe Analytics的自定义事件和维度中捕获的信息。 |
| 合并策略支持 | 您现在可以使用简单的下拉列表选择要应用于其区段定义的合并策略。 |

**错误修复**

* 修复了导致左边栏中属性和事件构建块加载缓慢的间歇性问题。
* 修复了导致估计器返回“NaN”响应的错误。
* 修复了某些字段打开错误规则构建画布的错误。

**已知问题**

* None.

有关详细信息，请参阅[分段服务概述](../../segmentation/home.md)。
