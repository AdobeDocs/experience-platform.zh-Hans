# Adobe Experience Platform 发行说明

**发布日期：2019 年 6 月 28 日**

Adobe Experience Platform中的新增功能：

* [[!DNL Data Science Workspace]](#dsw)
* [[!DNL Decisioning Service]](#decisioning)
* [[!DNL Query Service]](#query)

现有功能的更新：

* [[!DNL Experience Data Model (XDM)]](#xdm)
* [[!DNL Segmentation Service]](#segmentation)

## [!DNL Data Science Workspace] {#dsw}

Adobe Experience Platform [!DNL Data Science Workspace] 是一项完全托管的服务，位于 [!DNL Experience Platform] 它使数据科学家能够通过构建和运行机器学习模型，从Adobe解决方案和第三方系统中的数据和内容无缝地生成见解。 [!DNL Data Science Workspace] 紧密集成 [!DNL Platform] 并支持端到端的数据科学生命周期，包括XDM数据的探索和准备，然后开发和实施模型以自动丰富 [!DNL Real-Time Customer Profile] 使用机器学习分析。

**主要功能**

| 功能 | 描述 |
| -----------| ---------- |
| 资源调配和计算隔离 | 配置使数据科学家能够在中执行不受信任的代码所需的专用计算资源 [!DNL Experience Platform] 安全无虞。 |
| 首次用户体验 | 包括各种机器学习框架和语言的现成示例，例如 [!DNL Python]、R、PySpark和Scala [!DNL Spark]. |
| Notebooks | 面向数据科学家/数据工程师的定制环境，由 [!DNL Jupyter Notebooks] 以让他们能够准备数据、提取功能并使用精选的一系列库和流行的机器学习框架开发ML模型。 |
| 数据探索 | 无缝访问引入的XDM数据 [!DNL Platform] 集成 [!DNL Platform Data Access] SDK。 |
| 数据可视化 | 在中执行SQL查询的能力 [!DNL Jupyter Notebooks] 加快数据准备和功能设计。 |
| 功能管道 | 适用于Scala/PySpark的API/SDK可部署功能工程管道，以将核心XDM数据转换为功能架构。 |
| 模型创作 | 模板和运行时，使数据科学家能够专注于模型开发，而无需实施用于访问数据和计算资源的基础架构代码。 您可以导入模型代码并将其投入运行，以从中数据派生分析 [!DNL Platform]. |
| 企业模型管理 | 支持多租户数据模型以跟踪模型版本和相关超参数配置，为合作伙伴生态系统奠定基础。 |
| 模型评估 | 在中评估和优化回归和分类模型 [!DNL Python]、 PySpark 、 R和Scala。 |
| 模型部署 | 能够比较多个实验运行的评估指标和配置，并将最佳模型作为服务发布。 |
| 批次评分 | 扩充 [!DNL Real-Time Customer Profile] 使用机器学习分析或将它们作为数据集写回 [!DNL Platform] |
| 计划 | 与集成 [!DNL Platform] 编排服务通过API使用用户定义的计划自动执行模型训练、评分和功能管道。 |

**已知问题**

* 计划和功能管道目前只能通过API使用，UI将在未来版本中添加。

欲知更多信息，请访问 [数据科学工作区概述](../../data-science-workspace/home.md).

## [!DNL Decisioning Service] {#decisioning}

Adobe Experience Platform [!DNL Decisioning Service] 提供以下能力：以编程方式智能地从给定个人的一组可用选项中选择“下一个最佳体验”，将其交付到任何渠道或应用程序，以及执行报告和分析。

预建的丰富数据模型能够以与渠道无关的方式支持“下一个最佳优惠”决策的使用案例。

**主要功能**

| 功能 | 描述 |
| -----------| ---------- |
| 业务对象存储库 | JSON模式模型驱动的存储库允许开发人员创建、读取、更新和删除各种业务对象。 存储库提供通用的、表达式的查询API以及模式感知搜索。 |
| 存储库容器 | 在业务对象存储库中，开发人员可以将其关注点隔离在项目、业务或组织单位或项目生命周期阶段（例如，开发和集成、暂存或实时生产使用中）。 这些隔离称为存储库容器。 |
| 角色和权限 | 使用 [!DNL Admin Console]，组织可以创建和管理用户档案，以按类型、访问操作和容器授予对资源的定向访问权限。 可以将用户添加到这些访问配置文件中，并根据这些策略自动计算有效访问权限。 |
| 预建优惠对象模型 | 无需首先构建数据模型， [!DNL Platform] 开发人员可以利用预建的JSON架构和关系创建优惠目录、定义决策规则和约束，以及为决策收集优惠。 |
| 基于用户档案和非用户档案数据的决策规则 | 与 [!DNL Real-Time Customer Profile] 允许开发人员创建利用用户档案数据的决策规则。 不仅可以使用用户档案属性做出决策，而且还可以根据用户档案的体验事件历史记录以及基于与用户身份无关的业务实体（例如流量条件、产品库存）做出决策。 任意 [!DNL Experience Data Model] (XDM)实体，其中存在架构 [!DNL Schema Registry] 可用于决策规则。 规则是第一类实体，可重复用于任何决策选项和活动。 |
| 排名和上限 | 对满足给定用户的所有资格和其它约束条件的决策选项进行排名并选择最佳选项。 可以使用额外的每个用户和全局上限约束来限制可用选项的公开，从而可以在考虑资源约束和用户疲劳的情况下进行个性化。 |
| [!DNL Decisioning] REST API | 此 [!DNL Decisioning Service] 可使用简单的REST API调用，以获取给定个人的下一个最佳选件。 量度API可用于检查实时优惠建议和上限值。 |
| 将决策事件流式传输到 [!DNL Data Lake] 和 [!DNL Query Service] | 此 [!DNL Decisioning Service] 自动创建数据集以将所有XDM决策事件自动流入 [!DNL Data Lake]. 然后，可以使用数据集进行分析和报告 [!DNL Query Service]. |
| 开发人员支持 | 自助选择加入，其中包含有关Adobe I/O的文档，包括各种主题的教程。 |

**已知问题**

* 选件数据模型不会通过 [!DNL Schema Registry] 因此，只能以有限的方式加以推广。 模型架构具有允许附加自定义数据的内置结构。 将来，您将能够扩展基本XDM模型类来定义自己的自定义决策域。
* 您必须配置选件管理域模型，并且必须在此产品上下文中管理用户和集成。

## [!DNL Query Service] {#query}

[!DNL Query Service] 提供了在Adobe Experience Platform中使用标准SQL查询数据的能力，以支持多种不同的分析和数据管理用例。 它是一个无服务器工具，允许您在中连接任何数据集。 [!DNL Data Lake] 并将查询结果捕获为新的数据集以用于报告， [!DNL Data Science Workspace]，或进行摄取 [!DNL Profile Service].

您可以使用 [!DNL Query Service] 构建数据分析生态系统，创建消费者在各种互动渠道中的全景图。 这些渠道可能包括：

* 销售点系统
* Web
* 移动设备
* CRM系统

**主要功能**

| 功能 | 描述 |
| -----------| ---------- |
| 查询编辑器 | 使用基于Web的工具编写、验证、测试和执行查询。 它包括一个控制台，用于获取有关查询执行的详细信息以及预览查询结果的功能。 |
| 数据集创建 | 创建数据集 [!DNL Experience Platform] 通过标准SQL语法。 |
| Adobe定义的函数 | 将快捷方式功能用于识别会话或设置归因等常见任务。 |
| BI工具连接 | 使用在常见BI工具中找到的PostgreSQL (Postgres)驱动程序连接到 [!DNL Query Service] 以创建报告和可视化图表。 支持的工具包括： [!DNL Tableau]， [!DNL Power BI]、和 [!DNL Looker]. 在“凭据”选项卡上查找身份验证信息。 |
| 数据库管理工具连接 | Connect [!DNL Aqua Data Studio] 或 [!DNL DB Visualizer] 到 [!DNL Query Service] 用于数据探索和数据集创建功能。 [!DNL Query Service] 还支持来自R Studio的连接。 在“凭据”选项卡上查找身份验证信息。 |
| 命令行查询工具 | 连接PSQL以便能够从命令行运行查询。 |
| 查询日志 | 保留查询执行的历史记录 [!DNL Query Service] 使您能够查找以前的SQL以用于编辑、执行或从结果中创建数据集。 |
| 查询计划API | 通过此API计划定期执行的查询。 |

**已知问题**

* [!DNL Query Editor] 显示了100行查询结果的示例。 要保留完整的结果集，请使用查询日志中的数据集创建功能。
* 近期版本将添加对视图的支持以及用于向查询应用计划的UI。

有关详情，请参阅 [!DNL Query Service]，请参见 [产品文档](../../query-service/home.md).

## [!DNL Experience Data Model] (XDM) {#xdm}

标准化和互操作性是背后的关键概念 [!DNL Experience Platform]. [!DNL Experience Data Model] (XDM)由Adobe驱动，致力于标准化客户体验数据并定义用于客户体验管理的架构。

XDM是一个公开记录的规范，旨在提高数字体验的强大功能。 它为任何应用程序提供通用结构和定义，以便与Adobe Experience Platform上的服务进行通信。 通过遵守XDM标准，所有客户体验数据都可以纳入到通用表示中，从而以更快、更集成的方式提供见解。 您可以从客户操作中获得有价值的见解，通过区段定义客户受众，并将客户属性用于个性化目的。

XDM是一种机制，它允许 [!DNL Experience Cloud](由Adobe Experience Platform提供支持)，可在正确的时间通过正确的渠道将正确的信息传达给正确的人。

用于以下项的方法： [!DNL Experience Platform] 已构建， [!DNL XDM System] 可操作 [!DNL Experience Data Model] 架构供使用 [!DNL Experience Platform] 组件。

**新增功能**

| 功能 | 描述 |
| ---------- | ------------ |
| JSON架构约束 | 以下数据类型现在在用户界面中具有用于定义约束的其他选项： `string`  — 最小/最大长度、模式、默认值、格式（如中定义） [JSON架构draft-6](https://tools.ietf.org/html/draft-wright-json-schema-01))和 `double`  — 最小值/最大值，默认值。 |
| 自定义 `$id` | 现在，您可以提供自己的 `$id` 在POST请求中创建资源时的值。 |
| 架构注册表性能改进 | 优化了合并模式生成和增强模式缓存，从而大大缩短了API响应时间。 |

**错误修复**

* 已将identityMap字段移出上下文/个人资料并放入其自己的架构字段组中，以使定义身份更加直观。
* 已基于上下文/个人资料修补所有现有架构，并带有context/identitymap。
* 修复了未提供版本时的错误消息。
* 修复了以下错误： [!DNL Schema Registry] 为配置文件合并架构调用提供随机响应。
* 修复了合并架构在中未显示正确字段的错误 [!DNL Schema Registry].
* 修复了有时无法使用有效的命名空间创建身份描述符的错误。
* 修复了对象使用时的取消引用问题 `properties` 而不是 `allOf`.

**已知问题**

* 无法扩展 [!DNL Platform]通过添加字段而提供的字段组。
* 从架构组合中删除字段组时，描述符未删除。
* 无法创建没有标签的枚举字段。

要了解有关使用XDM的更多信息，请使用 [!DNL Schema Registry] API和 [!DNL Schema Editor]，请阅读 [XDM系统文档](../../xdm/home.md).

## [!DNL Segmentation Service] {#segmentation}

[!DNL Segmentation Service] 从您的个人资料存储中定义个人资料的特定子集，描述区分您的个人资料存储中可销售的人员的标准。 区段可以基于代表客户与您的品牌接触点的记录数据（如人口统计信息）或时间序列事件。

例如，在专门针对跑鞋的电子邮件营销活动中，您可以使用过去30天内搜索跑鞋但未完成购买的所有用户的受众区段。 另一个示例可能是使用区段来定位网站内容，以便该区段仅显示给属于奖励计划特定层的访客。

**新增功能**

| 功能 | 描述 |
| -----------| ---------- |
| 相对时间规则 | 您现在可以选择滚动时间窗口，例如14天前、3到5小时前等。 |
| XDM字段摘要 | 对于左边栏上的属性，现在可以使用摘要查看基础数据。 |
| 左边栏搜索 | 改进了左边栏区段部分的搜索功能。 |
| eVar友好名称 | 改进了对友好名称的支持，让您能够更轻松地查看在自定义事件和维度中从Adobe Analytics捕获的信息。 |
| 合并策略支持 | 您现在可以使用简单的下拉列表选择要应用于其区段定义的合并策略。 |

**错误修复**

* 修复了导致左边栏中的属性和事件构建块加载缓慢的间歇性问题。
* 修复了导致估算器返回“NaN”响应的错误。
* 修复了某些字段打开不正确的规则构建画布的错误。

**已知问题**

* None.

欲了解更多信息，请参见 [分段服务概述](../../segmentation/home.md).
