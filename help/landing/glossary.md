# Adobe Experience Platform 术语表 {#adobe-experience-platform-glossary}

## 同类群组

**访问控制**:基于角色的访问控制使管理员能够向Experience Platform用户分配访问权限和权限。权限包括视图和／或使用Experience Platform功能的能力，如创建沙箱、定义模式和管理数据集。

**访问密钥ID**:访问密钥ID是与S3秘密访问密钥关 [!DNL Amazon] 联的唯一标识符。访问密钥ID和秘密访问密钥被结合使用来签署[!DNL Amazon Web Services](AWS)请求。

**操作**:在中 [!DNL Platform Launch]，操作是一种特定类型的规则组件，它定义在发生事件并评估和传递条件后应发生的情况。

**激活**:激活是指用户将区段或用户档案映射到目标（如、或） [!DNL Oracle Eloqua]所 [!DNL Google]执行的操 [!DNL Salesforce Marketing Cloud]作。

**活动**:在 [!DNL Offer Decisioning]中，活动包含通知优惠选择的逻辑。

**管理员**:您组织中的一个或多个个人可以配置和自定义在Adobe Admin ConsoleExperience Platform的权限。

**Adobe Admin Console**:Adobe Admin Console为管理Adobe产品授权和访问您的组织提供了一个中心位置。通过控制台，管理员可以授予用户组对各种平台功能(如“管理数据集”、“视图数据集”或“管理用户档案”)的访问权限。

**Adobe Experience Platform**:Adobe Experience Platform在整个企业内实现数据和内容标准化，为实时消费者用户档案提供强大动力，支持数据科学并加快内容速度，从而在客户旅程中推动体验个性化。

**Adobe Experience Platform Launch**: [!DNL Platform Launch] 是一个标签和SDK管理生态系统，与Experience Platform和应用程序 [!DNL Experience Cloud] 集成。[!DNL Platform Launch] 提供部署、统一和管理分析、营销和广告集成的工具，这些工具是在所有客户端设备上提供相关客户体验所必需的。

**Adobe Experience Platform查询服务**:使数据分析师能够查询事件和用户档案，用于分析和机器学习。借助查询服务，数据科学家和分析师可以抽取存储在Experience Platform中的所有数据集(包括行为数据、销售点(POS)、客户关系管理(CRM)等)，并查询这些数据集以回答有关数据的特定问题。

**Adobe Experience Platform细分服务**:支持构建细分并根据实时客户用户档案数据生成受众。然后，这些受众可以导出到数据湖中自己的数据集。

**Adobe智能服务**:Attribution AI和客户人工智能等智能服务是基于机器学习和人工智能的模型，这些模型旨在构建并需要Experience Platform运行和操作。

**Adobe I/O**:Adobe I/O是Experience Platform的一部分，它提供对开发人员集成、扩展和自定义平台所需的一切的访问，包括API、事件、开发人员控制台和有用的工具。

**Adobe Sensei**:Adobe Sensei是支持Experience Platform的情报框架。它还提供一套人工智能服务，使品牌能够增强其提供实时、个性化客户体验的能力。

**AmazonS3桶**: [!DNL Amazon S3] 存储段是存储在生态系统中的数据的基本 [!DNL Amazon] 容器。存储段包含对象，每个对象都使用由开发人员指定的唯一密钥进行存储和检索。

**AmazonS3连接器**:S3 [!DNL Amazon] 连接器使Experience Platform客户能够安全地连接和访问其 [!DNL Amazon] S3数据。

**附加保存策略**:“追加”保存策略是指定第三方数据通过连接进行收录，并在数据集末尾附加任何新数据或行时使用的选项。以前摄取的行保持不变，只将自上次调度运行后创建的行引入Experience Platform。 在源系统中更改的任何行在Experience Platform上都保持不变。

**阵列**:数组用于具有相同数据类型的有序元素。

**人工智能**:人工智能是计算机系统的理论和发展，能够执行通常需要人类智能的任务，如视觉感知、语音识别、决策和语言之间的翻译。

**属性**:属性是指表示用户档案的特征。

**属性合并**:当使用实时客户用户档案API定义合并策略时，该对 `attributeMerge` 象指示在发生数据冲突时合并策略优先处理用户档案属性的方式。它等效于在平台UI中定义合并策略时选择[!UICONTROL 合并方法]。

**Attribution AI**: [!DNL Attribution AI] 是由Adobe Sensei提供支持的智能服务，可在整个客户生命周期内提供算法多渠道归因功能。

**受众**:受众是生成的一组符合区段定义标准的用户档案。

**受众大小**:受众大小是满足细分定义标准并有资格成为受众会员的用户档案总数。

**受众快照**:受众快照可捕获分段时符合区段条件的所有用户档案。

## B

**回填**:对于计划的源，回填选项支持获取历史数据。

**回填期**:回填期是设置通过源连接接收第三方历史数据的时间长度的选项。选择“永远”的回填期将收集源数据的整个历史记录给Experience Platform。

**批**:批是在一段时间内收集的一组数据，并作为单个单元一起处理。数据集由多批组成。

**批ID**:批处理ID是Adobe生成的批量数据标识符。

**批量摄取**:批处理摄取允许您将数据作为批处理文件引入Experience Platform。批量是由一个或多个要作为单个单位摄取的文件组成的数据单位。

**批分段**:批量细分是当前数据选择流程的替代方法，它通过细分定义一次性移动所有用户档案数据以生成相应的受众。创建后，将保存并存储此段，以便导出以供使用。

**构建**:在中 [!DNL Platform Launch]，内部版本是包含执行库中包含的业务逻辑所需的所有配置和代码的文件或文件集，允许您在网站或移动应用程序上部署该库。

**商业智能工具**:商业智能(BI)工具主要与集成在一起 [!DNL Experience Platform Query Service]。BI工具是从内部和外部系统收集和处理大量非结构化数据的应用程序软件类型。

## C

**限制**:在 [!DNL Offer Decisioning]中，在决策规则中使用上限（也称为频率上限）来定义优惠的显示次数。有两种类型的上限：在合并的优惠受众（称为“全局上限”）中，可以建议多少次优惠，以及向同一最终用户建议多少次用户档案(称为“目标上限”)。

**目录**:在源和目标的上下文中，目录是一个图库，可与Adobe应用程序和第三方技术建立连接。请勿与[!DNL Catalog Service]混淆。

**[!DNL Catalog Service]**: [!DNL Catalog Service] (有时称 [!DNL Catalog]为)是Adobe Experience Platform内数据位置和谱系的记录系统。当所有引入Experience Platform的数据作为文件和目录存储在数据湖中时，[!DNL Catalog]会保留这些文件和目录的元数据和说明，以便进行查找、监控和数据管理。

**类**:在体验数据模型(XDM)中，类定义用于构建模式的最小字段集，并定义模式所表示的业务对象的基本行为。

**客户端**:客户端是通过PostgreSQL协议或HTTP API连 [!DNL Query Service] 接到的外部工具或应用程序。

**集合**:在中 [!DNL Offer Decisioning]，集合是基于由营销人员定义的预定义条件(如优惠的类别)的优惠子集。

**与PII营销活动结合使用**:将任何个人可识别信息(PII)与匿名数据相结合的营销行动。从广告网络、广告服务器和第三方数据提供商获取的数据合同通常包括特定合同禁止使用具有直接可识别数据的数据。

**命令行界面**:命令行界面是基于文本的工具，可用于连接到以执 [!DNL Query Service] 行原始查询。

**合成**:组合是组件的分组，这些组件一起构成模式。

**条件**:在 [!DNL Platform Launch]中，条件是计算必须返回或的逻辑语句的规则 `true` 组件 `false`。在执行对规则执行任何操作之前，所有条件都必须评估为`true`，所有例外条件都必须评估为`false`。

**控制台**:在 [!DNL Query Service]中，控制台提供有关查询的状态和操作的信息。控制台将显示连接状态至[!DNL Query Service]、正在执行的查询操作以及这些查询产生的任何错误消息。

**合同(“C”)标签**:合同(“C”)数据使用标签用于对具有合同义务或与客户数据管理策略相关的数据进行分类。

**C1合同标签**:合同 `C1` 数据使用标签指定数据只能以聚集形式从Adobe Experience Cloud导出，而不包括个人或设备标识符。例如，源自社交网络的数据。

**C2合同标签**:合 `C2` 同数据使用标签指定无法导出到第三方的数据。一些数据提供者在其合同中有条款禁止从最初收集数据的地方出口数据。 例如，社交网络合同通常会限制您从它们收到的数据的传输。 C2比C1更具限制性，C1只需要聚合和匿名数据。

**C3合同标签**:合同 `C3` 数据使用标签指定不能与直接可识别信息组合或以其他方式使用的数据。一些数据提供商在合同中有条款禁止将数据与直接可识别的信息合并或使用。 例如，从广告网络、广告服务器和第三方数据提供商获取的数据合同通常包括使用直接可识别数据的特定合同禁止。

**C4合同标签**:合同 `C4` 数据使用标签指定数据不能用于定位任何现场或跨站点的广告或内容。C4是限制性最强的标签，因为它包含C5、C6和C7标签。

**C5合同标签**:合同 `C5` 数据使用标签指定数据不能用于基于兴趣的内容或广告的跨站点定位。满足以下三个条件时，即发生基于兴趣的定位或个性化：现场采集的数据用来推断用户的兴趣；用于其他上下文，如其他站点或应用程序。并用于根据这些引用选择提供哪些内容或广告。

**C6合同标签**:合同 `C6` 数据使用标签指定数据不能用于现场广告定位。现场广告定位包括选择和投放您组织的网站或应用程序上的广告，或衡量此类广告的投放和效果。 这包括使用先前收集的有关用户兴趣的现场数据来选择广告、处理有关显示广告的数据、显示广告的时间和位置以及用户是否采取了与广告相关的任何操作，如选择广告或购买。

**C7合同标签**:合 `C7` 同数据使用标签指定数据不能用于内容的现场定位。现场内容定位包括选择和投放您组织的网站或应用程序上的内容，或衡量此类内容的投放和有效性。 这包括先前收集的有关用户选择内容的兴趣、处理有关显示内容、显示频率或时间、显示时间和位置以及用户是否采取了与内容相关的任何操作（例如选择内容）的数据。

**C8合同标签**:合 `C8` 同数据使用标签指定数据不能用于测量贵组织的网站或应用程序。这不包括基于兴趣的定位，即您使用此服务后来在其他上下文个性化内容和／或广告的信息集合。

**C9合同标签**:合同 `C9` 数据使用标签指定数据不能用于数据科学工作流。一些合同明确禁止用于数据科学的数据。 有时，这些术语的用词是禁止将数据用于人工智能(AI)、机器学习(ML)或建模。

**C10合同标签**:合同 `C10` 数据使用标签指定数据不能用于拼接标识激活。某些数据使用策略限制使用拼接标识数据进行个性化。 如果区段的合并策略使用“专用图”选项，则`C10`标签会自动应用到这些区段。

**创建日期列**:通过源连接指定第三方数据时，可选择“创建日期”列。当选择附加保存策略且数据集模式包含多个日期字段时，您必须从可用模式中选择以指定创建日期键列。 选择覆盖保存策略后，“创建日期”选项将不可用。

**以选择方式创建表**:Create Table as Select(CTAS)是SQL命令，作为完整有效SQL查询的一部分执行时，该命令将指 [!DNL Query Service] 示将查询结果保留在数据集中。可以创建新结果集、覆盖以前的结果或追加到以前的结果。

**跨站点数据**:跨站点数据是来自多个站点的数据的组合，包括现场数据和非现场数据的组合，或来自多个非现场源的数据的组合。

**跨站点定位营销活动**:使用数据进行跨站点广告定位的营销活动。来自多个站点的数据的组合，包括现场数据和非现场数据的组合或来自多个非现场源的数据的组合，称为跨站点数据。 跨站点数据通常被收集和处理，以推断客户的兴趣。

**自定义身份命名空间**:您的组织可以创建自定义身份命名空间来代表特定组织或业务案例的身份。

**自定义标签**:自定义数据使用标签允许您创建特定标签并将其应用到满足特定业务需求的数据字段。

**客户人工智能**:客户AI是由Adobe Sensei提供支持的智能服务，它通过基于AI的倾向丰富客户用户档案，并支持客户细分和定位工作。

## D

**数据字典**:在 [!DNL Platform Launch]中，数据字典（也称为数据映射）是在属性中定义的一组数据元素。

**数据元素**:在 [!DNL Platform Launch]中，数据元素是在规则和扩展中使用的指针，用于指向客户端设备上存在的特定数据。

**数据获取**:数据摄取是将数据从源添加到Experience Platform的过程。数据可以通过多种方式引入平台，包括流、批处理或通过源连接器添加。

**数据层**:在 [!DNL Platform Launch]中，数据层是客户端设备上存在的数据结构，其中包含有关查看页面或屏幕的上下文的元数据。

**数据管理**:数据治理包括用于确保数据符合有关数据使用的法规和组织政策的战略和技术。

**数据集成合作伙伴**:数据集成合作伙伴无需编写代码即可简化大量数据从200多个源到Experience Platform的加载和转换，并实现自动化。

**数据集标签**:数据使用标签可以添加到数据集。该数据集中的所有字段都将继承该数据集的标签。

**数据科学工作区**: [!DNL Data Science Workspace] 在Experience Platform中，客户能够利用跨平台和Adobe应用程序的数据创建机器学习模型，从而创建智能细分、生成洞察并提供预测，从而大大增强最终用户的数字体验。

**数据源**:数据源是用户指定的来源数据。数据源的示例包括移动应用程序、用户档案和／或体验事件、网站用户档案事件或CRM。

**数据管理**:数据管理者是负责管理、监督和执行组织数据资产的人员。数据管理人员还确保数据管理政策得到保护并得到维护，以符合政府法规和组织政策。

**数据流**:数据流是共享相同模式并由同一源发送的消息集或集合。

**数据类型**:数据类型是可重用的XDM资源，它定义在分层表示中包含多个属性的对象类型字段。

**数据使用标签**:数据使用标签允许您对反映隐私权相关考虑事项和合同条件的数据进行分类，以符合法规和公司政策。添加到数据集的数据使用标签会继承下来或应用到该数据集中的所有字段。 数据使用标签也可以直接应用于字段。

**数据流**:数据流是从源流到目标流入平台的虚拟数据管线。

**数据流运行**:数据流运行是根据用户指定的Experience Platform进入计划的数据流。

**数据集**:数据集是存储和管理构建，用于模式集合，通常是表，其中包含（列）和字段（行）。

**数据集ID**:Adobe生成的摄取数据集的标识符。

**数据集输出**:数据集输出提供了一种机制，用于确定“将表创建为选择”选项用于特定运 [!DNL Query Service] 行。

**增量列**:增量列允许您选择源数据字段来表示增量摄取的时间戳。

**增量保存策略**:增量保存策略是通过源连接接收第三方数据的选项。该选项允许用户指定新的或更改的源数据行被引入Experience Platform。 将新行添加到数据集的末尾，并在Experience Platform的数据集中更新更改的行。

**描述符**:在体验数据模型(XDM)中，描述符是一组附加的与模式相关的元数据，用于描述字段的特定行为。Experience Platform可以使用描述符来了解预期的模式行为，如两个模式之间的关系。

**目标**:目标是任何端点(如Adobe应用程序、广告平台、云存储服务或营销服务)的一般术语，受众在此处被激活和交付。

**目标类别**:目标类别是具有相似特性的目标的分组。

**目标目录**:目标目录是列表中可用目标的Experience Platform。

**直接呼叫规则**:在 [!DNL Platform Launch]中，直接调用规则是从页面直接调用时执行的规则，绕过事件检测和查找系统。

**显示名称**:在体验数据模型(XDM)中，显示名称是UI中显示的字段的用户友好名称。

## E

**合格优惠**:符合条件的优惠可以一致地提供给用户档案，因为它满足上游定义的约束。

**合格规则**:在 [!DNL Offer Decisioning]中，合格规则应用于与日历、计划和限制约束相关的用户档案。

**电子邮件定位营销活动**:使用电子邮件定位活动中数据的营销操作。

**嵌入代码**:在 [!DNL Platform Launch]中，嵌入代码是放置在站点或环境的HTML中的脚本标签。嵌入代码指示浏览器在何处检索内部版本。

**明细列表**:明细列表（枚举）是一个XDM字段，它受限于一组预定义值。

**环境**:在 [!DNL Platform Launch]中，环境是一组部署指令，它指定构建的主机投放和文件格式。库必须与环境配对，才能构建。

**错误诊断**:错误诊断允许为摄取的批生成详细的错误消息。错误阈值允许您在批处理失败之前配置可接受错误的百分比。

**事件**:在 [!DNL Platform Launch]中，事件是特定类型的规则组件，它是在客户端设备上发生的用于开始执行规则的触发器。

**事件实体**:在事件建模的上下文中，事件实体表示与客户可以采取的操作、系统或您希望跟踪随时间变化的任何其他概念相关的概念。属于此类别的实体应由基于[!DNL XDM ExperienceEvent]类的模式表示。

**事件**:事件是与用户档案关联的行为数据。

**体验模式模型** [!DNL Experience Data Model] (XDM)(XDM)是一个开放源代码框架，它使用标准统一数据，以便与Experience Platform和Adobe Experience Cloud应用程序一起使用。XDM实现数据结构的标准化，并加快速度并简化从大量数据中获取洞察的过程。

**实验**:实验是通过用实时生产数据的样本部分训练实例来创建训练模型的过程。这与针对未结合测试数据集测试的经过训练的模型不同。 这也不同于某些机器学习框架中的实验概念，它实际上意味着示例建模项目。

**体验事件**:体验事件表示发生与客户体验相关的交互或事件时系统的快照。体验事件是不可改变的事实记录，它代表所发生的事情，而不进行聚合或解释。 在体验数据模型(XDM)中，此概念由[!DNL XDM ExperienceEvent]类捕获。

**扩展**:在 [!DNL Platform Launch]中，扩展是添加到属性的功能 [!DNL Platform Launch] 包。扩展通常以特定营销或分析解决方案为中心，提供将该技术部署到客户环境所需的工具。

**标准身份命名空间**:标准身份命名空间是由Adobe提供的预定义身份命名空间，代表行业标准解决方案，通常用于识别客户。

**流摄取**:流式摄取允许您将数据从客户端和服务器端设备实时发送到Experience Platform。

**流细分**:流细分是一种持续不断的数据选择过程，它可根据用户活动更新细分。在构建并保存区段后，会将区段定义应用于传入数据至[!DNL Real-time Customer Profile]。 会定期处理细分添加和删除，确保您的目标受众保持相关性。

**系统视图**:系统视图是源数据集的可视表示形式，这些数据集流 [!DNL Real-time Customer Profile] 向目标。

## T

**时间序列数据**:时间序列数据提供记录主体直接或间接采取操作时系统的快照。

**经过培训的模型**:训练的模型表示模型——训练过程的可执行输出，其中一组训练数据被应用到模型实例。经过培训的模型将保留对从其创建的任何智能Web服务的引用。 经过训练的模型适合于评分和创建智能Web服务。

**令牌**:令牌是一种双因素身份验证安全性类型，可用于授权使用计算机服务 [!DNL Query Service]。
