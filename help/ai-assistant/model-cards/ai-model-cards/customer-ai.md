# 客户人工智能倾向打分模型卡

作为[Adobe Experience Platform中的智能服务](../../../intelligent-services/home.md)的一部分，您可以使用[Customer AI](../../../intelligent-services/customer-ai/overview.md)在单个级别生成客户预测和解释。

在影响因素的帮助下，您可以使用客户人工智能来告诉您客户可能会做什么以及为什么。 此外，您还可以从客户AI预测和洞察中获益，通过提供最合适的服务和消息使客户体验个性化。

请阅读此模型卡，了解用于支持客户AI的AI模型的信息。

## 模型概述 {#model-overview}

* Customer AI是AI支持的模型，旨在根据用户过去的行为以及与企业的交互为用户提供倾向性分数。 使用客户AI预测客户采取特定操作（例如购买、使用内容或搅动）的可能性。 此模型部署在Experience Platform中，并与各种营销和客户分析工作流程集成。
* 该模型旨在通过预测客户将执行给定操作（例如购买、注册订阅或参与电子邮件营销活动）的概率，为营销人员和客户参与团队提供可操作的见解。 这些输出允许企业优化受众分段并根据预测的行为个性化客户互动。
* 这是一个&#x200B;**监督学习分类模型**，它根据历史客户数据预测事件（购买、改动、参与）发生的概率。 利用梯度提升决策树(GBDT)结合Logistic回归对倾向性得分进行训练。
* 此模型的主要用户是营销专业人员、数据分析师和客户参与团队，他们利用Experience Platform推动数据驱动型营销策略。
* 客户人工智能直接集成到Experience Platform的人工智能服务中，允许用户通过预建的API和仪表板访问模型输出。 模型生成的倾向分数可在[Adobe Journey Optimizer](https://experienceleague.adobe.com/en/docs/journey-optimizer/using/get-started/get-started)和[Real-Time CDP](../../../rtcdp/home.md)中使用，以优化受众细分和定制营销策略。

## 预期用途 {#intended-use}

* 此模型主要用于&#x200B;**客户细分、目标营销和流失预测**。 企业利用该模型来&#x200B;**预测客户购买意图、优化营销活动以及增强个性化努力**。 例如，一家电子商务公司可能会使用该模型来识别有意向的购物者，并为他们提供独家促销。
* 营销人员经常在&#x200B;**确定适合的客户**&#x200B;和&#x200B;**优化参与工作**&#x200B;方面遇到困难。 此模型&#x200B;**通过提供数据驱动的方法来确定客户目标，从而减少猜测**，确保营销资源得到高效分配。
* 该模型适用于多个行业，包括电子商务、零售、金融服务、电信和媒体。 任何依赖于客户参与和个性化营销的业务都可以从此模型中受益。
* 模型&#x200B;**不应用于高风险决策**，例如财务信用评分、医疗诊断或法律评估。 此外，由于潜在的伦理问题，它不适用于预测个人敏感行为（健康状况、政治偏好）。

## 模型输入和输出 {#model-inputs-and-outputs}

* 该模型可处理客户行为数据、人口统计属性和历史交互。 这包括网站访问频率、过去的购买历史记录、与营销电子邮件的接触情况以及人口统计信息等数据。
* 输入数据必须结构化为包含客户属性和行为信号的JSON对象。 对于批量处理，模型会接受根据Experience Platform的数据摄取标准设置格式的CSV文件。
* 该模型输出介于0和1之间的倾向分数，其中较高的值表示预测事件发生的可能性较高。 此外，它提供了功能重要性分数，使用户能够了解哪些因素影响了预测。

**示例输入**

```json
{
  "customer_id": 12345,
  "past_purchases": 3,
  "last_visit_days": 7,
  "email_click_rate": 0.4
}
```

**示例输出**

```json
{
  "customer_id": 12345,
  "propensity_score": 0.82
}
```

## 训练数据

* 模型已接受有关通过Experience Platform收集的&#x200B;**第一方匿名客户交互数据**&#x200B;的培训。 它包括跨多个行业的历史客户&#x200B;**行为数据、交易记录、电子邮件交互和参与量度**。
* 训练数据集包含来自多种Experience Platform客户集的&#x200B;**100万条客户记录**。 这些记录包括&#x200B;**历史客户互动、交易数据、行为参与日志和人口统计信息**，这些信息来自零售、电子商务、电信和金融等不同行业。 收集了24个月的数据，确保充分体现季节性趋势和长期参与模式。
* 数据集主要来自高参与度用户，这可能会导致选择偏差。 为了缓解这种情况，模型采用了分层采样、偏差审计技术和数据补充策略。
* 数据集会经历广泛的预处理，以确保数据一致性、质量和可用性。
   * **处理缺少的值**：使用平均值计算（用于数字字段）、模式计算（用于分类字段）和预测建模（用于复杂缺少的情况）的组合处理缺少的值。
   * **分类编码**：通过单热编码和目标编码技术将分类变量（如客户区段和购买类别）转换为数字表示形式。
   * **功能缩放和标准化**：最小最大缩放适用于有界变量（年龄、收入），而z分数标准化适用于常分布功能。
   * **其他预处理**：管道包括异常检测和删除、重复过滤、时间戳标准化和功能工程以增强预测建模。

## 模型架构和培训

* 模型使用&#x200B;[!DNL XGBoost]**利用**[!DNL Gradient Boosting Decision Trees] (GBDT)，针对结构化数据进行了优化。 它根据历史客户事件序列进行培训，以识别预测性行为模式。
* 该模型采用有监督学习方法，以[!DNL XGBoost]作为主要学习算法，采用GBDT方法建立。 此外，将Logistic回归作为基准预测准确性的基准模型纳入。
* 已使用&#x200B;**[!DNL TensorFlow]**、**[!DNL XGBoost]**&#x200B;和&#x200B;**[!DNL scikit-learn]**&#x200B;开发模型。 使用&#x200B;**[!DNL NVIDIA V100]GPU**&#x200B;在AdobeAI云基础架构上运行培训，支持大规模数据集。
* [!DNL NVIDIA V100 GPUs]，已接受Google Cloud基础架构培训。
* [!DNL AUC-ROC]、精确召回和交叉验证。

## 绩效和评估

* 模型使用保持性验证方法进行测试，80%的数据用于训练，20%保留用于评估。
* 模型的有效性使用[!DNL AUC-ROC] (0.85)、精确召回率(0.78)和F1分数(0.80)来衡量。 这些量度有助于评估模型在不同区段之间的预测能力。
* 历史数据有限的新客户区段的准确性较低。
* 对于历史数据有限（冷启动问题）的客户，此模型的表现可能会不佳。 此外，季节性影响（节假日购物趋势）可能需要频繁的再培训以保持准确性。

## 公平和偏见

* 模型已进行&#x200B;**人口统计对等测试**&#x200B;和&#x200B;**对抗公平性评估**，以检测不同用户区段之间的性能差异。
* 分析显示具有低历史交互数据&#x200B;**的用户性能下降了** 5%。 为了解决这个问题，该模型在训练过程中采用了重新加权技术。
* 对数据集进行分层以确保不同客户人口统计的比例表示，并在训练过程中引入公平性约束以防止模型偏向任何特定组。 利用人口均等分析进行定期的偏差审计，以便在发现业绩差异时进行调整。

## 解释性和可解释性

* 该模型利用&#x200B;**[!DNL SHapley Additive Explanations](SHAP)**&#x200B;来量化每个输入特征对其预测的影响，从而提供客户属性如何影响倾向分数的透明度。 SHAP值支持全局可解释性，确定所有预测中最具影响力的因素，也支持局部可解释性，解释针对特定客户的单个预测。
* 该模型支持&#x200B;**[!DNL Local Interpretable Model-Agnostic Explanations](LIME)**&#x200B;和SHAP，可提供输入特征如何影响预测的见解。 LIME通过创建输入数据的扰动版本和观察预测的变化来生成局部解释，而SHAP为每个特征分配贡献值，提供模型决定的全局和局部解释。

## 鲁棒性和泛化性

* 在未查看的数据集上测试时，模型保持80%的[!DNL AUC-ROC]，表明新客户记录具有很强的泛化能力。 在不同客户群中，性能保持稳定，但当用户行为明显偏离历史模式时，性能会略有下降。
* 该模型已根据受干扰和对立输入（包括丢失数据、异常注入和故意错误标记）进行评估。 虽然在正常情况下性能仍然稳健，但在极端对抗修改下发现精度略有下降（约3-5%）。

## 安全和隐私注意事项

* 模型&#x200B;**不会处理或保留任何个人身份信息(PII)**，并且所有用于培训的数据都会进行匿名化处理和聚合。 它坚持严格遵守GDPR、CCPA和内部Adobe隐私政策，以确保负责任地使用数据。
* 该模型结合了差分隐私技术，将受控噪声添加到数据中，防止个人被重新识别。 此外，在模型训练和推理之前，还使用&#x200B;**散列、匿名化和标记化方法移除PII**。

## 部署和集成

* 该模型托管在Adobe Experience Platform AI服务上，并与各种Adobe应用程序集成。 它通过API端点提供，允许无缝访问以实现跨营销和客户参与工作流的实时预测和批量处理。
* 该模型在基于&#x200B;**[!DNL Kubernetes]的部署**&#x200B;上运行，具有自动缩放功能以高效地处理各种工作负载。 计算资源包括[!DNL NVIDIA V100]个GPU用于训练，以及优化基于CPU的推理以实现实时可扩展性。

## 监控和维护

* 通过&#x200B;[!DNL WatsonX]**对模型进行连续**&#x200B;监控，跟踪精度漂移、特征重要度漂移和预测稳定性等关键性能指标。 异常检测和警报机制会在发生与预期行为显着偏离时通知团队。
* 使用更新的客户交互数据每月对模型进行重新培训，以确保持续的相关性。 定期重新培训有助于缓解数据漂移和季节性波动，这些波动可能影响预测准确性

## 伦理问题和负责任的人工智能

* 如果监测不当，该模型可能会给决策带来偏差。 例如，如果培训数据中某些人口统计数据的比例过高，则模型可能会不公平地偏袒特定客户群体。
* Experience Platform遵循负责任的人工智能准则，确保在部署之前对模型进行&#x200B;**偏差审核、公平性测试和人为监督**。

## 已知限制

* 在历史数据不足的情况下，该模型可能难以准确预测新发布产品或客户细分的结果。 此外，如果在重新培训期间未考虑客户行为的季节性变化，则可能会导致预测准确度出现波动。
* 当客户历史记录稀疏时（例如对于首次购买者或参与数据最少的用户），性能会下降。 此外，如果客户行为&#x200B;**由于外部因素（如经济下滑或行业趋势**）而发生转变，则模型可能需要快速调整以保持准确性。

## 未来的改进

* 未来的迭代将包括转移学习技术，以提高冷启动用户的性能，并增强对不断变化的客户行为的适应性。 此外，还将引入实时数据集成，以提高动态营销环境中的模型响应速度和准确性。
